## Architecture

Single-activity Jetpack Compose app (minSdk 24, targetSdk 36). No ViewModel, no Navigation Compose вАФ screen state lives in `MainActivity.kt` via `rememberSaveable`.

**Current structure:**
- `MainActivity.kt` вАФ entry point; uses `HorizontalPager` (`rememberPagerState`) for swipe tab navigation; `NavigationBar` items call `pagerState.animateScrollToPage()`; `showScan: Boolean` via `rememberSaveable` drives the scan overlay; **NFC reader-mode** infra lives here (`NfcAdapter.ReaderCallback`): a lazy `NfcAdapter`, `enableReaderMode(this, this, READER_FLAGS, null)` in `onResume` / `disableReaderMode` in `onPause`, `READER_FLAGS = FLAG_READER_NFC_A|B|F|V or FLAG_READER_SKIP_NDEF_CHECK`; `AndroidManifest.xml` declares `<uses-permission android:name="android.permission.NFC" />` and `<uses-feature android:name="android.hardware.nfc" android:required="false" />` вАФ `required="false"` keeps the app installable on NFC-less devices, with `NfcState.NoHardware` gracefully disabling bind affordances; `@Volatile var onTagScanned: ((String) -> Unit)? = null` is a nullable hook invoked on the main thread (`mainHandler.post`) with `normalizeNfcUid(tag.id)`; `@Volatile var onTagForChipInfo: ((Tag) -> Unit)? = null` is the **highest-priority** hook for the debug ¬Ђ–Ш–љ—Д–Њ –Њ —З–Є–њ–µ¬ї (GET_VERSION) flow; `@Volatile var onTagForMark: ((Tag) -> Unit)? = null` is the ¬Ђ–Ю—В–Љ–µ—В–Є—В—М –Ъ–Я¬ї scan hook (also raw `Tag`, since marking needs `readChipCode` + `unlock` off the uid); `@Volatile var onTagForProvision: ((Tag) -> Unit)? = null` is the admin bulk chip-provisioning hook (raw `Tag`, since binding needs to bind + write onto the tag), armed by `ProvisioningScreen`; `@Volatile var onTagForVerify: ((Tag) -> Unit)? = null` is the admin read-only chip-verification hook (raw `Tag`, since verification needs `readChipCode` + `LegendCrypto.bid` off the tag вАФ no write, no bind), armed by `CheckChipScreen`. `onTagDiscovered`'s priority is **`onTagForChipInfo` вЖТ `onTagForProvision` вЖТ `onTagForVerify` вЖТ `onTagForMark` вЖТ `onTagScanned` вЖТ idle**: each posts to the main thread and returns; if none is set, the idle foreground path recognizes our own chips and opens ¬Ђ–Ю—В–Љ–µ—В–Є—В—М –Ъ–Я¬ї: it reads `(code, uid)` once (`readChipCode(tag)` over raw NfcA + `normalizeNfcUid(tag.id)`), and if `code != null` (a –Ъ–Я chip) **or** `uid in boundUidsSnapshot` (a bound team-member bracelet) publishes a `CapturedScan(code, uid, trustedClock.sample())` to `nfcLaunchScan` (the `TimeSample` is snapshotted on the binder thread at tap time) вАФ else drops the tag silently (see ¬ЂChip-scan opens ¬Ђ–Ю—В–Љ–µ—В–Є—В—М –Ъ–Я¬ї¬ї below). The `(code, uid)` read here is reused downstream вАФ no second chip read. `onTagForProvision`/`onTagForVerify` never co-arm (the admin home opens one sub-overlay at a time), so their relative order is cosmetic. All hooks are `@Volatile` because `onTagDiscovered` reads them from the binder thread while `DisposableEffect` writes them from the main thread. Provisioning (under `showAdmin`/`showProvisioning`) and the debug chip-info flow (under `showSettings`) are never armed simultaneously, but the distinct hook keeps each `DisposableEffect` owning exactly one hook. `AndroidManifest.xml` declares `android:launchMode="singleTop"` on `MainActivity` вАФ **kept harmlessly** (it originally backed `onNewIntent` NDEF re-delivery, now gone with the raw-chip migration). **The NDEF cold/warm-launch path is removed** (raw `K24` chips carry no NDEF): there is no `NDEF_DISCOVERED` intent filter, no `handleNfcIntent`, and no `CapturedScan(openOnly = true)` publish anymore вАФ the ¬Ђ–Ю—В–Љ–µ—В–Є—В—М –Ъ–Я¬ї overlay opens only via the live reader-mode path (`onTagDiscovered` while the app is foreground) or the orange FAB. (Trade-off accepted: tapping a –Ъ–Я chip with the app fully closed no longer auto-launches it.) `nfcState: NfcState { NoHardware, Disabled, Available }` (top-level enum) is recomputed each `onResume` (`adapter == null`вЖТNoHardware, `!isEnabled`вЖТDisabled, else Available) and read by composables. The bind-chip flow (see ¬ЂBind-chip overlay¬ї below) registers/clears `onTagScanned` via `DisposableEffect`; the debug ¬Ђ–Ш–љ—Д–Њ –Њ —З–Є–њ–µ¬ї flow registers/clears `onTagForChipInfo` via its own `DisposableEffect` (DEBUG-only вАФ armed via `onReadChipInfo`, reads `readChipVersion(tag)` off-main and surfaces `chipModelFromVersion(...)`). `SelectedTeamState { None, Loading, Missing, Present(team) }` private sealed interface drives the ¬Ђ–Ъ–Њ–Љ–∞–љ–і–∞¬ї tab: `produceState` switches from `Loading` (initial) to `None`/`Present`/`Missing` as `observeTeam` emits вАФ `Loading` suppresses the empty-state flash before Room delivers the first value; `todayIso()` (the shared top-level helper in `data/DateUtils.kt`, `SimpleDateFormat` not `java.time.LocalDate` for minSdk-24/no-desugaring compatibility) is passed down to `CompPickerScreen`. `onRaceSelected` fires an **on-tap prefetch** before opening the team picker: `container.applicationScope.launch { teamRepo.refreshTeams(raceId) }` + `legendRepo.refreshLegend(raceId)` (applicationScope, NOT composition scope, so the warm-up outlives the closing comp picker) вАФ the network starts during the screen transition. `TeamPickerScreen.onRefresh = teamRepo::refreshTeams` stays the authoritative UI-state driver; the rare duplicate GET with the on-tap prefetch is accepted (idempotent/ETag-guarded)

- `data/api/AppSignatureInterceptor.kt` вАФ OkHttp interceptor; HMAC-SHA256 signs every request (canonical string from `docs/API.md`), adds the six `X-App-*`/`X-Install-Id` headers; `ts`/signature recomputed per call (survives retries). Pure top-level helpers live here: `buildCanonical(method, fullPath, ts, bodyHash)` (4-part `\n`-joined canonical вАФ the 4th part is the **body hash**), `sign(secret, canonical)`, `sha256Hex(bytes)` (lower-hex), and the `EMPTY_BODY_SHA256` constant (hex SHA-256 of an empty body, byte-identical to hashing empty bytes so GET/empty-body signing is unchanged). `intercept` computes the body hash by buffering the built request (`okio.Buffer` + `request.body?.writeTo(buffer)` вЖТ `sha256Hex`); a **null body, a one-shot body (`isOneShot()`), or an `IOException` while buffering all fall back to `EMPTY_BODY_SHA256`** (wrapped in try/catch so signing can never crash). A `nowSeconds: () -> Long = { System.currentTimeMillis() / 1000 }` param is the `ts` source: `AppContainer` passes `{ trustedClock.signingSeconds() }` so a skewed wall-clock doesn't blow the server's ¬±300 s window (see ¬ЂTrusted time¬ї below). **Self-heal retry-once (GET/HEAD only):** after `proceed()`, if `request.method in ("GET","HEAD") && response.code == 403 && nowSeconds() != usedTs` it `close()`s, re-signs with a fresh `ts`, and `proceed`s once more вАФ because the inner `ServerTimeInterceptor` already re-anchored from the `403`'s `Date` header, the second attempt is signed with trusted time and clears the clock-skew `403`. **POST is never auto-retried** (`login`/`logout`/`bindTag`): the server returns a generic `403` indistinguishable from a real auth failure, and replayability вЙ† idempotency вАФ POST self-heals on the next user-initiated action (signing is already trusted). - `data/api/ServerTimeInterceptor.kt` вАФ OkHttp interceptor that re-anchors `TrustedClock` from the HTTP `Date` response header on every **network** response (free, backend-free server-time source вАФ no protocol change). Snapshots `elapsed()` on both sides of `proceed()`; `rtt = elapsedAfter вИТ elapsedBefore`; pins the parsed `Date` epoch to an **overflow-safe RTT-corrected midpoint** `anchorElapsed = elapsedBefore + rtt / 2` (not `(before+after)/2`, which could overflow `Long`). Re-anchoring is gated so only a genuinely live `Date` is trusted: **network only** (`response.networkResponse != null` вАФ a cache hit's `Date` is stale), **RTT in `0..maxRttMs`** (default 10 s; a **negative** RTT is a clock/timing anomaly and an over-long RTT makes the midpoint too coarse вАФ both dropped), and **`Date` present+parseable** (`headers.getDate("Date")` вЖТ `null` for missing/malformed leaves the anchor untouched). Trust is **structural**: installed only on the single-host `/app/` client (`BuildConfig.API_BASE_URL`/HTTPS), so no `request.url.host` gate вАФ KDoc records that a second host on this client would require adding one. On accept calls `onServerTime(serverMs, anchorElapsed, wallNow, bootNow)`; ordering/out-of-order handling lives in `TrustedClock`. **Tested by `ServerTimeInterceptorTest`** (a trust boundary, so the ¬Ђadapters untested¬ї convention is waived): network `Date` вЖТ `onServerTime` with parsed epoch + midpoint; cache hit / missing / malformed `Date` вЖТ no call; RTT 10 000 accepts, 10 001 rejects, negative rejects; out-of-order passes each response's own midpoint through.

- `data/time/ClockAnchorStore.kt` вАФ plain-SharedPreferences store for the `ClockAnchor` (trusted-clock warm-start seed); mirrors the `ThemePreference`/`InstallId` pattern (pure injected `load: (String) -> String?` + `save: (String, String?) -> Unit` seam + `fromSharedPreferences(context)` adapter, synchronous read). **Atomic write (P1):** the anchor is one **delimited string under a single key** `"anchor"` (`"$serverEpochMs|$anchorElapsedMs|$capturedWallMs|${bootCount ?: ""}"`, empty 4th segment = `bootCount == null`), not four keys вАФ a process kill between `apply()`s can't leave a half-written mix. `read(): ClockAnchor?` (null if key absent or malformed: wrong field count / non-numeric segment), `write(anchor)` (one `save` вЖТ one `apply()`), `clear()`. Adapter: prefs file `"kolco24.clock"`. Tested by `ClockAnchorStoreTest` (round-trip incl. `bootCount` non-null/null, `clear` removes the key, pre-seeded read, absent/malformed вЖТ null).

**Scan overlay pattern:**
`MainActivity` wraps `Scaffold` in a `Box(Modifier.fillMaxSize())`. The `ScanScreen` is rendered after the Scaffold inside the same Box вАФ when `showScan = true` it covers the entire screen. Dismiss via the close button or system back (handled by `BackHandler`) sets `showScan = false`. No navigation component used. The overlay is fed `roster = teamForTab?.members`, `bindings = Map<String,Int>` (`scanBindings`: `nfcUid вЖТ numberInTeam`, inverted from the binding rows but **filtered to the current roster's `numberInTeam` slots** вАФ `rosterSlots`), `nfcAvailable`, and an `onScanTag: suspend (ScanInput, Long) -> ScanEvent`. Filtering is a correctness guard: a binding left over from a member since removed from the roster would otherwise let a stale chip substitute for an un-scanned participant and falsely complete a take (`complete` only compares `present.size` to `expectedCount`); excluding it makes the stale chip read as `UnboundChip`. The scan FAB is **gated on `teamForTab != null`** вАФ with no resolved team (or a selection whose team row went missing) `onScanClick` routes to the team picker instead of opening an empty scan, and `onScanTag` also refuses (`BadKp`) when `scanRoster.isEmpty()`, so a missing-team selection (pointer non-null, team row gone) can't open a `expectedCount = 0` orphan take. `onScanTag` runs the impure side over a `ScanInput`: `ScanInput.Live(tag)` reads `code = readChipCode(tag)` (on `Dispatchers.IO`) + `uid = normalizeNfcUid(tag.id)`, while `ScanInput.Captured(code, uid)` (the opening tap, already read on the binder thread / launch intent) uses them directly вАФ **no second chip read**. The shared tail (`legendRepo.unlock` when a code is present вЖТ `classifyTag(...)` вЖТ persist via `MarkRepository`) and the tap-time `now` threading are identical for both, returning the `ScanEvent` for the screen's `reduce`. A private `ScanTakeState` (plain class, `remember(showScan)` so it's fresh per overlay open) mirrors the UI session on the DB side: `markId`/`checkpointId`/`expectedCount`/`buffer`/`lastScanAt`. A `Kp` event starts a new take (new UUID) when the window expired (`now вИТ lastScanAt > SCAN_WINDOW_MS`), no take is open, or the –Ъ–Я id changed вАФ else it just re-stamps; a `Member` before any –Ъ–Я goes to `buffer`, after into `MarkRepository.addMember`. Persisting writes run on `container.applicationScope.async { ... }.await()` (so they outlive the closing overlay yet still hand the id back for the in-session `addMember` chain). The `marks` flow (`remember(selectedTeamId){ markRepo.observeMarks(it) ?: flowOf(emptyList()) }.collectAsState`) feeds `MarksScreen`.

**Chip-scan opens ¬Ђ–Ю—В–Љ–µ—В–Є—В—М –Ъ–Я¬ї (open-on-tap + auto-close):**
Scanning a –Ъ–Я chip or a bound team-member bracelet **opens** the scan overlay and feeds that first tap into the session (previously the overlay opened only via the orange FAB; an idle –Ъ–Я tap just showed a Toast). With the raw-chip migration there is now a **single open source вАФ the live-idle foreground path** (the NDEF cold/warm-launch sources are gone): `onTagDiscovered`'s idle branch reads `(code, uid)` once and, if `code != null` (a –Ъ–Я chip) `|| uid in boundUidsSnapshot` (a bound bracelet), publishes `CapturedScan(code: ByteArray?, uid: String, sample: TimeSample)` (top-level in `MainActivity.kt`; `code == null` вЗТ member bracelet; `sample` is `trustedClock.sample()` snapshotted at tap time, replacing the old `capturedAt: Long`) to the Activity-level `val nfcLaunchScan = MutableStateFlow<CapturedScan?>(null)`. (The old `openOnly` flag and the `handleNfcIntent` cold/warm publish were removed вАФ raw chips can't cold-launch the app at all.) Two `@Volatile`/flow Activity fields support the binder-thread path: `boundUidsSnapshot: Set<String>` (the selected team's bound uids, mirrored by a host `LaunchedEffect(scanBindings) { activity.boundUidsSnapshot = scanBindings.keys }` вАФ a **coarse open-gate** that may briefly lag a team switch; the authoritative roster-filtered `scanBindings` still governs scoring, so a stale bracelet just reads `UnboundChip`), and `pendingScan: MutableStateFlow<CapturedScan?>` (the opening tap handed to `ScanScreen` to drain). The host **dispatcher** is a `LaunchedEffect(captured, teamState)` (NOT a bare `collectAsState` + inline `if` вАФ side effects must not run during composition, and keying on `teamState` re-fires the `Loading вЖТ Present` deferral) branching on `SelectedTeamState`: another overlay up (`showScan || teamFlowStep != None || confirmTeamId != null || showSettings || showAdmin || showProvisioning || showCheckChip || bindSlot != null || unbindSlot != null`) вЖТ drop + clear (buffering into `pendingScan` only for the narrow `showScan`-true-but-hook-not-yet-armed race); `Present` вЖТ reset all overlays (same as `onScanClick`) + `pendingScan.value = scan` + `showScan = true` + clear; `None`/`Missing` вЖТ open the comp picker (`pickerRaceId = selectedRaceId; teamFlowStep = CompPicker`) + clear; `Loading` вЖТ **return without clearing** so the next `teamState` transition re-runs the effect (a defensive guard for a live-idle tap landing mid team-switch вАФ the original cold-launch race it solved is gone with the NDEF launch path). `ScanScreen` **drains** `pendingScan` via a continuous collect in its arm-`onTagForMark` `DisposableEffect` (see the `ui/scan/ScanScreen.kt` bullet) as a `ScanInput.Captured` with the captured `sample` (its monotonic `elapsedMs` = tap time, not drain time, so a slightly-later drain doesn't corrupt the 20 s window math; its `trustedMs`/`bootCount` persist into the take). **Auto-close** lives in `ScanScreen`: window expiry calls `currentOnClose()` after the finalize, and completion (`isComplete(session, roster.size)`) shows a ~1 s green ¬Ђ–У–Њ—В–Њ–≤–Њ!¬ї beat then closes.

**Trusted time (TrustedClock вАФ clock-skew defense for take times):**
The take time of a –Ъ–Я used to be raw wall-clock (`System.currentTimeMillis()`), so reanchoring the phone's clock ¬Ђmoved¬ї the recorded time. `TrustedClock` (`data/time/TrustedClock.kt`) instead derives a **trusted** time anchored to the monotonic `SystemClock.elapsedRealtime()` timer plus server time from the HTTP `Date` header вАФ `trusted = serverEpochMs + (elapsedNow вИТ anchorElapsedMs)`. A wall-clock change can't move it (the monotonic timer isn't settings-adjustable); only a **reboot** breaks the anchor (`elapsedRealtime` resets), detected by monotonic regression and/or `Settings.Global.BOOT_COUNT` mismatch. **Zero backend changes** вАФ the `Date` header is free server time. Wiring: `ServerTimeInterceptor` (inner OkHttp interceptor, see its bullet) re-anchors on every network response; `ClockAnchorStore` persists the anchor (single delimited key) for a true warm start (same boot session вЗТ trusted time available immediately on a process restart, no network); `AppContainer` reads `BOOT_COUNT` once per process and owns the `TrustedClock`/store lazies. A take now records five times via `TimeSample` вЖТ `MarkRepository.startKpTake`: `trustedTakenAt` (scoring + order), `takenAt`/`updatedAt` (raw wall), `elapsedRealtimeAt` + `bootCount` (monotonic mark + boot session for future ќФ forensics) вАФ see `MarkEntity` (Room v10) and `MarkDao.observeForTeam`'s `COALESCE(trustedTakenAt, takenAt)` ordering. The **scan window** (`SCAN_WINDOW_MS`, ticker, expiry) runs on the monotonic `elapsedMs`, immune to clock changes. **Request signing** uses trusted time (`AppSignatureInterceptor.nowSeconds = { trustedClock.signingSeconds() }`): if the wall-clock is >¬±300 s off the sync would `403`; instead the inner `ServerTimeInterceptor` anchors even from the `403`'s `Date`, and the signature interceptor's GET/HEAD retry-once re-signs with trusted `ts` вЖТ `200` вАФ the app **self-heals** while the skew banner keeps nagging ¬Ђ–њ–Њ–њ—А–∞–≤—М—В–µ —З–∞—Б—Л¬ї. UI: `ClockWarningBanner` (global, `Skewed` only) + `ScanClockBanner` (`Skewed` accent + soft `NoSync` in the scan overlay). A `MainActivity` `LaunchedEffect` ticks `recomputeStatus()` every 5 s so skew surfaces without a network event.

**GPS track recording (foreground service + dual batch upload + Precise/Economy profiles):**
A team records its race track locally with manual start/stop, in the background with the screen off, on one of two **profiles** (`TrackProfile`, persisted via `trackProfilePreference`): `Precise` (GPS HIGH_ACCURACY, 15 s, default вАФ today's behavior) or `Economy` (HIGH_ACCURACY, 180 s вАФ battery mode toggled from a Settings Switch; the 3-min interval duty-cycles the radio for a several-fold saving). `TrackRecordingService` (`TrackRecordingService.kt`, **foreground service `type=location`**, `START_NOT_STICKY` so a system kill does **not** silently resume вАФ restart is UI-only) reads `raceId`/`teamId` from intent extras, **re-checks** `ACCESS_FINE/COARSE_LOCATION` first thing in `onStartCommand` (TOCTOU: a revoke between the launcher and start вЖТ `stopSelf()` without ever calling `startForeground` with the location type, which would `SecurityException` on target 34+), then `startForeground(FOREGROUND_SERVICE_TYPE_LOCATION)` with a low-importance ¬Ђ–Ч–∞–њ–Є—Б—М —В—А–µ–Ї–∞¬ї notification (¬Ђ–Ш–і—С—В –Ј–∞–њ–Є—Б—М —В—А–µ–Ї–∞ ¬Ј N —В–Њ—З–µ–Ї¬ї + a ¬Ђ–°—В–Њ–њ¬ї action `PendingIntent` вЖТ `ACTION_STOP` вЖТ `stopSelf`). It reads `activeProfile = container.trackProfilePreference.profile.value` and starts the engine via a private synchronous `startEngine(profile)` (`val e = LocationEngineFactory.create(this, profile); engine = e; e.start(onPoints, onError)` вАФ starts the **captured local**, NPE-safe against a concurrent `engine = null`; does **not** pre-stop, callers own the stop). **Segment minting:** a `private var segmentId: String?` field holds one UUID per recording session; on the fresh-start path in `onStartCommand` it is set via the pure top-level `nextSegmentId(current, wasTearingDown, mint)` (mints a new id when `current == null` **or** `wasTearingDown`, else reuses вАФ so a duplicate/idempotent start intent keeps the segment, a stopвЖТstart gap or a teardown-replace gets a fresh one; JVM-tested by `SegmentIdTest`); `startEngine` snapshots it into a local `s` (defensively minting if null, alongside the `r`/`t` race/team snapshot) and passes it into the `onPoints` lambda; `finishTeardown()` resets `segmentId = null` so the next session mints fresh. The **profile soft-restart** (`profileJob`/`flushThen`) never touches `segmentId`, so a mid-race PreciseвЖФEconomy toggle stays one segment. Each `RawFix` batch вЖТ `applicationScope.launch { trackRepository.insertAll(fixes, r, t, s) }` + `trackRecordingState = Recording(teamId, count)`; `onError` (provider lost / `SecurityException`) tears down so it never hangs as a fix-less ¬ЂRecording¬ї. **Live-apply (mid-race profile toggle):** a `profileJob` is **(re)launched inside `onStartCommand`** (next to `countJob`, **not** once вАФ `teardown()` cancels `serviceScope`, a later start recreates it, so a once-launched job would die on the new scope) on `Dispatchers.Main.immediate` (so `engine` writes serialize with the main-thread lifecycle + flush callbacks); e.stop(); restart to the LATEST .value }`. **`flushThen(e, after)`** runs `after` **exactly once** (`AtomicBoolean`) fired by the flush callback **or** a `mainHandler.postDelayed(complete, FLUSH_TIMEOUT_MS вЙИ 4 s)` fallback (a never-completing `flushLocations()` can't leave the ¬Ђ–°—В–Њ–њ¬ї service stuck foreground);
