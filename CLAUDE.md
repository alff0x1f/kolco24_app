# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build commands

```bash
./gradlew assembleDebug              # build debug APK
./gradlew assembleRelease            # build release APK
./gradlew lintDebug                  # run lint (must pass before merging)
./gradlew testDebugUnitTest          # run unit tests
./gradlew connectedDebugAndroidTest  # run instrumented tests (requires emulator/device) — guards Room migration
```

## Architecture

Single-activity Jetpack Compose app (minSdk 24, targetSdk 36). No ViewModel, no Navigation Compose — screen state lives in `MainActivity.kt` via `rememberSaveable`.

**Current structure:**
- `MainActivity.kt` — entry point; uses `HorizontalPager` (`rememberPagerState`) for swipe tab navigation; `NavigationBar` items call `pagerState.animateScrollToPage()`; `showScan: Boolean` via `rememberSaveable` drives the scan overlay; `SelectedTeamState { None, Loading, Missing, Present(team) }` private sealed interface drives the «Команда» tab: `produceState` switches from `Loading` (initial) to `None`/`Present`/`Missing` as `observeTeam` emits — `Loading` suppresses the empty-state flash before Room delivers the first value; `todayIso()` (the shared top-level helper in `data/DateUtils.kt`, `SimpleDateFormat` not `java.time.LocalDate` for minSdk-24/no-desugaring compatibility) is passed down to `CompPickerScreen`. `onRaceSelected` fires an **on-tap prefetch** before opening the team picker: `container.applicationScope.launch { teamRepo.refreshTeams(raceId) }` + `legendRepo.refreshLegend(raceId)` (applicationScope, NOT composition scope, so the warm-up outlives the closing comp picker) — the network starts during the screen transition. `TeamPickerScreen.onRefresh = teamRepo::refreshTeams` stays the authoritative UI-state driver; the rare duplicate GET with the on-tap prefetch is accepted (idempotent/ETag-guarded)
- `ui/marks/MarksScreen.kt` — Отметки tab; orange FAB triggers ScanScreen overlay via `onScanClick` callback
- `ui/legend/LegendScreen.kt` — Легенда tab; stateless: `LegendScreen(checkpoints: List<CheckpointEntity>, hasTeam, onChooseTeam, modifier)` renders two states — `!hasTeam` → **LegendNoTeam** (map glyph + «Выбрать команду» → `onChooseTeam`, same callback the Команда/Отметки empty states use); else the CP list (**LegendList**) fed from Room. The old `LegendLocked` "ДО СТАРТА" card and the `legendVisible` param are **gone** — the legend is now always returned and locked checkpoints are masked per-row instead (the server-level `is_legend_visible` flag was removed). `CheckpointRow` branches on `cp.cost == null` → **masked** (Lock glyph in the label slot, mono `?-${number.padStart(2,'0')}`, subtitle «Откроется на КП», muted `onSurfaceVariant`, no check icon); else the open/revealed row. `ScoreCard` sums known costs only (`mapNotNull { it.cost }`), and a `lockedCount = checkpoints.count { it.cost == null }` drives a quiet «+N закрытых КП» hint under the progress bar (shown only when `> 0`). Bold dark FilterChips, no trailing icon for unvisited CPs. The mono row label (`<стоимость>-<номер>`) pads `number.toString().padStart(2, '0')` so `5-01` doesn't regress to `5-1`. The `taken`-based dim+strike styling is retained though `taken` is always `false` this iteration (no marks feature yet)
- `ui/team/TeamScreen.kt` — Команда tab; state-hoisted (`selectedTeam: TeamEntity?`/`category`/`teamMissing` collected in `MainActivity`), `null`/missing → `TeamEmptyContent`, otherwise hero card (`startNumber` + name + «Категория X · N человек») and roster from `team.members`. There is **no** standalone «Сменить команду» row anymore (it moved into the Settings overlay, matching `assets/template_v2.html`); the «Прочее» `SectionCard`'s `MiscRow` is now clickable (`onClick` + `.clickable`), and «Настройки» (subtitle «Сменить команду») fires `onOpenSettings` while «Справка» stays inert (default `{}`)
- `ui/settings/SettingsScreen.kt` — Settings (Настройки) full-screen overlay; stateless `SettingsScreen(onBack, onChangeTeam, modifier)` — a `Column` with a `TopAppBar` (back arrow → `onBack`) and one inset «Команда» `surfaceContainerLow` card whose single `ChangeTeamRow` (charcoal `SwapHoriz` avatar, «Сменить команду» / «Выбрать другую команду соревнования», `ChevronRight`) → `onChangeTeam`. The row is a local copy of the old `SwitchTeamRow` (duplication accepted per design; no shared file)
- `ui/teampicker/` — team-selection flow (screens 04/04b/04c/04d): `CompPickerScreen` (pick a race), `TeamPickerScreen` (registered-teams list with search), `TeamSwitchSheet` (`ModalBottomSheet` confirm), `TeamEmptyContent` (empty/missing state), `TeamPickerLogic.kt` (pure, unit-tested helpers: `raceStatusPill`, `splitRaces`, `filterTeams`, `teamToken`, `displayTeamName`, `initials`, `peopleLine`; ISO-date strings compared lexicographically — no `java.time.LocalDate`, minSdk 24 without desugaring). `TeamPickerScreen` is **non-blocking on a cold cache**: instead of a centered `CircularProgressIndicator`, it always renders the `LazyColumn` (with `CompContextCard`) and shows a slim `LinearProgressIndicator` flush under the `TopAppBar` (wrapped in a `Column`, so it does not scroll away) while `load == PickerLoad.Loading`; the "Пока никто не зарегистрирован" empty text is gated on `load != Loading` so a cold open never flashes a false "nobody registered". Existing empty/`Offline`/`HttpError`/`Forbidden`/retry + `staleCache`/`forbidden` snackbar branches take over once loading finishes with no data
- `ui/scan/ScanScreen.kt` — full-screen overlay for NFC checkpoint marking (A3 screen)
- `ui/theme/Color.kt` — full M3 light/dark token set; cool-grey light palette; `OrangeCta = #C65A2E`
- `ui/theme/Theme.kt` — `Kolco24Theme`; always uses the static brand palette (`LightColorScheme`/`DarkColorScheme`); no dynamic color wiring
- `ui/theme/Type.kt` — typography; `RobotoMono` (`FontFamily`, `res/font/roboto_mono_bold.ttf`) is used directly for all mono-number labels (КП row labels including the masked `?-NN`, score tokens). The M3 `Typography` object still uses the platform default for prose text.

**Data layer** (races + teams + legend sync — signed `GET` with ETag, Room as single source of truth):
- `Kolco24App.kt` — `Application`; builds `AppContainer`. On `onCreate`, Launch A fires `refreshRaces()` fire-and-forget then **prefetches the nearest race**: reads `raceRepository.races.first()`, computes `nearestRaceId(races, todayIso())`, and (if non-null) warms it via concurrent child `launch { teamRepository.refreshTeams(nearest) }` + `legendRepository.refreshLegend(nearest)` (logs results; `nearest == null` on offline/empty → no-op, no crash). Launch B `collectLatest`s `teamRepository.selectedTeam` and on each non-null `raceId` refreshes **both** `legendRepository.refreshLegend(raceId)` and `teamRepository.refreshTeams(raceId)` concurrently via `supervisorScope { launch {} launch {} }` (`collectLatest` + `supervisorScope` so a team switch cancels both in-flight fetches before starting new ones, with sibling failures isolated from each other; the persisted selection emits immediately on subscribe, so a cold start refreshes exactly once). All warm-up runs on `container.applicationScope`. Registered via `android:name=".Kolco24App"`. NB: the root composable in `MainActivity` is `Kolco24AppRoot()` (renamed to avoid clashing with this class).
- `AppContainer.kt` — manual DI (no Hilt); lazy `installId`, `OkHttpClient` + `AppSignatureInterceptor`, `Json`, `ApiClient`, `AppDatabase`, `RaceRepository`, `TeamRepository`, `LegendRepository`, plus `applicationScope = CoroutineScope(SupervisorJob() + Dispatchers.IO)`. Reads secrets from `BuildConfig`.
- `data/api/AppSignatureInterceptor.kt` — OkHttp interceptor; HMAC-SHA256 signs every request (canonical string from `docs/API.md`), adds the six `X-App-*`/`X-Install-Id` headers; `ts`/signature recomputed per call (survives retries). Pure `buildCanonical`/`sign` helpers live here.
- `data/api/ApiClient.kt` — generic `FetchResult<out T>` (`Success<T>(data, etag)`/`NotModified`/`Forbidden`/`Error(code)`; non-success branches are `FetchResult<Nothing>`). `fetchRaces(etag): FetchResult<List<RaceDto>>`, `fetchTeams(raceId, etag): FetchResult<TeamsResponse>`, and `fetchLegend(raceId, etag): FetchResult<LegendResponse>` (`GET /app/race/$raceId/legend/`) all delegate to one private `conditionalGet(url, etag, parse)`; GET on `Dispatchers.IO`, `If-None-Match` sent verbatim with quotes, 10s timeouts. `IOException`/`SerializationException` → `Error(null)` (never throws).
- `data/api/dto/RacesResponse.kt`, `data/api/dto/TeamsResponse.kt`, `data/api/dto/LegendResponse.kt` — `@Serializable` DTOs with snake_case `@SerialName`; `Json { ignoreUnknownKeys = true }`. `TeamDto.startNumber` (`@SerialName("start_number")`) is nullable with a default — the backend added it recently and `docs/API.md` does not document it yet. `RacesResponse`/`RaceDto` no longer carries `is_legend_visible` (removed server-side along with the per-CP encryption rework). `LegendResponse(race: Int, checkpoints: List<CheckpointDto>, tags: List<TagDto> = emptyList())`; the legend is **always returned** now. `CheckpointDto(id, number, cost: Int?, type, description: String?, enc: EncDto?)` is flat (no `@SerialName` needed) — a **locked** CP arrives with an `enc:{iv,ct}` envelope (`EncDto(iv, ct)`, both Base64) **instead of** `cost`/`description` (so both are nullable; `enc != null` is the locked sentinel). `TagDto(bid, point, @SerialName("check_method") checkMethod, iv: String? = null, ct: String? = null)` maps each NFC tag's `bid → point`; `iv`/`ct` hold the unlock bundle envelope for tags that open locked CPs, `null` for open-CP tags. `type` (`start|finish|test|kp`) is currently ignored (no per-type styling). Crypto contract spec lives in `docs/design/API.md`.
- `data/db/` — Room **v4** (`AppDatabase`, `exportSchema = true` → `schemas/.../{1,2,3,4}.json` committed, no `fallbackToDestructiveMigration` — a schema version bump without a migration will crash on upgrade). v1 tables: `RaceEntity` (table `races`, server `id` PK — also the domain model, no third layer; **no** legend-visibility column anymore — `is_legend_visible` was dropped in v4), `SyncMetaEntity` (table `sync_meta`, composite PK `(origin, resource)`, stores ETag per origin/resource). v2 adds team-selection tables: `CategoryEntity` (`categories`; `CategoryDto.order` field maps to `sortOrder` column — `order` is a reserved SQL word, `MIGRATION_1_2` uses `sortOrder` accordingly), `TeamEntity` (`teams`, `@Index` on `raceId`; `members: List<TeamMemberItem>` stored as a JSON column via `TeamMembersConverter` `@TypeConverter`), `SelectedTeamEntity` (`selected_team`, single row PK `id = 1`; `SelectedTeamDao.observe(): Flow<SelectedTeamEntity?>` emits `null` when the table is empty — this is the sentinel for "no team chosen yet"). v3 adds `CheckpointEntity` (`checkpoints`, `@PrimaryKey id` = server CP id, `@Index` on `raceId`; doubles as the model). v4 (legend-encryption rework) makes `CheckpointEntity.cost`/`description` **nullable** and adds `locked: Boolean`, `encIv: String?`, `encCt: String?` (a locked CP carries the `enc` envelope instead of plaintext; `taken: Boolean = false` unchanged), and adds `TagEntity` (`tags`, `@PrimaryKey bid`, `@Index` on `raceId` and `point`; columns `point`, `checkMethod`, `iv: String?`, `ct: String?`) with `TagDao` (mirrors `CheckpointDao`: `observeTagsForRace`, `getByBid`, `@Transaction replaceAllForRace`). `CheckpointDao` mirrors `TeamDao`: `observeCheckpointsForRace(raceId)` (`ORDER BY number, id`) + `@Transaction replaceAllForRace` — the latter is now **preserve-on-resync (option A)**: it snapshots prior revealed (`cost IS NOT NULL`) rows, wipes+reinserts the server rows, then re-applies the prior plaintext to any incoming still-`locked` id via `reveal(id, cost, description)`, so a `200` refresh never re-locks a checkpoint the user unlocked offline; open rows arrive with content and overwrite cleanly. Each `MIGRATION_*` is raw `CREATE TABLE`/`CREATE INDEX` whose SQL must match Room's generated schema JSON exactly (column names stay camelCase, e.g. indices `index_teams_raceId`, `index_checkpoints_raceId`, `index_tags_raceId`, `index_tags_point`) — `assembleDebug`/KSP does **not** verify this; a mismatch only surfaces at runtime when the upgraded DB opens. `MIGRATION_2_3` only adds the `checkpoints` table + index. `MIGRATION_3_4` **recreates** `checkpoints` (copy-rename via a `*_new` temp table: nullable `cost`/`description` + `locked=0`/enc null for old rows) and `races` (drop `isLegendVisible`) — SQLite-24 can neither relax `NOT NULL` nor `DROP COLUMN` — and `CREATE TABLE tags` + its two indices; the recreate copies preserve all existing race/checkpoint data. An instrumented `MigrationTest` (`app/src/androidTest/`, `MigrationTestHelper` + `androidx.room:room-testing`) guards each migration; an instrumented `CheckpointDaoTest` guards preserve-on-resync (real Room `@Transaction` — Robolectric isn't on the classpath); run both via `./gradlew connectedDebugAndroidTest` (needs an emulator/device). DAOs and the converter's wiring are not unit-tested (trivial); the pure converter round-trip is (`TeamMembersConverterTest`).
- `data/RaceRepository.kt` — `races: Flow<List<RaceEntity>>` from Room; `refreshRaces(): RefreshResult` (`Updated`/`NotModified`/`Offline`/`Forbidden`/`HttpError`). On `Success`: `replaceAll(races)` **then** `upsert(etag)` as two separate transactions — order is deliberate, a crash between them keeps fresh data with a stale ETag so the next refresh re-fetches and self-heals. ETag upsert is skipped when the response omits the `ETag` header (null), leaving any previous ETag in place.
- `data/TeamRepository.kt` — per-race teams/categories + the selected team. `teamsForRace(raceId)`/`categoriesForRace(raceId)`/`observeTeam(teamId)`/`selectedTeam` are Room flows; `refreshTeams(raceId): RefreshResult` mirrors `refreshRaces` (ETag stored in `sync_meta` under resource `"race/$raceId/teams"`, replaceAllForRace **then** ETag upsert, ETag skipped when null); `selectTeam(raceId, teamId)` upserts the single `selected_team` row. DTO→entity mapping lives here.
- `data/LegendRepository.kt` — per-race checkpoint legend **and** NFC tags. `checkpointsForRace(raceId): Flow<List<CheckpointEntity>>` is a Room flow; `refreshLegend(raceId): RefreshResult` mirrors `refreshTeams` (ETag stored in `sync_meta` under resource `"race/$raceId/legend"`) and on `200` persists **both** arrays — `checkpointDao.replaceAllForRace(...)` then `tagDao.replaceAllForRace(...)` then ETag upsert (ETag skipped when null). `CheckpointDto.toEntity` sets `locked = (enc != null)`, `encIv/encCt = enc?.iv/ct`, cost/description pass through; `TagDto.toEntity` is 1:1. The repo takes `tagDao` + a `Json` (wired in `AppContainer`). `unlock(raceId, code: ByteArray): UnlockOutcome` is the **offline reveal path**: `LegendCrypto.bid(code)` → `tagDao.getByBid` (null → `Unknown`); an iv-null tag → `IdentityOnly(point)`; else it builds the `id→EncBlob` map from the race's checkpoints, calls `LegendCrypto.unlock(...)`, and maps the engine's `UnlockResult` → `UnlockOutcome` (`Revealed` persists each CP via `checkpointDao.reveal(id, cost, description)` — `locked`/`taken` untouched — and returns `Revealed(point, ids)`; `Failed`→`Failed`). Its only caller this iteration is the unit test (NFC scan wiring is deferred). `taken` is not in the legend API (it comes from NFC marks, not built yet) so it stays `false`.
- `data/crypto/LegendCrypto.kt` — pure, offline, Android-free crypto engine for the encrypted legend (mirrors the server `src/apps/mobile/crypto.py` + `legend_crypto.py`; spec in `docs/design/API.md`). `bid(code) = sha256(code).hex().take(16)`; `deriveWrapKey(code)` is hand-rolled HKDF-SHA256 over `HmacSHA256` (extract with 32 zero-byte `salt=None`, expand with `info="kp-wrap-v1"` US_ASCII, len 32); `open(key, ivB64, ctB64, aad)` is AES-256-GCM (`Cipher("AES/GCM/NoPadding")`, `GCMParameterSpec(128, iv)`, `updateAAD`, `doFinal`); `unlock(code, tag, encById, json): UnlockResult` does the `bundle_blob → content_key → enc` indirection (open bundle with `aad=bid`, then per cpId open `enc` with `aad=str(cpId)`), **never throws** (catches `GeneralSecurityException`/`SerializationException`/`IllegalArgumentException` → `Failed`). Consumes minimal value types (`UnlockTag`, `EncBlob`) — not DTOs/entities — so the repo builds the maps. **Base64 is okio `ByteString.decodeBase64()`**, not `java.util.Base64` (needs API 26 > minSdk 24) or `android.util.Base64` (stubbed in JVM tests). Result types `UnlockResult { Revealed, IdentityOnly, Failed }`, `RevealedCheckpoint`, `EncBlob`, `UnlockTag` are top-level in this file. A `LegendCryptoSanityTest` covers bid/wrap-key/round-trip/full unlock; `LegendCryptoTest` holds the server-vector assertions, `@Ignore`d until a server-generated fixture lands (the engine ships **crypto-unverified against the server** until then).
- `data/InstallId.kt` — pure `getOrCreate(load, save)` for the install UUID (≤64 chars, API requirement) over an injected key-value store; thin SharedPreferences adapter.
- `data/DateUtils.kt` — shared top-level helpers: `todayIso(): String` (`SimpleDateFormat("yyyy-MM-dd", Locale.US).format(Date())`, no `java.time` for minSdk-24/no-desugaring compatibility); `nearestRaceId(races, today): Int?` selects the soonest-starting current race (`effectiveEnd >= today`, then `minByOrNull { it.date }`; `null` when none current) — `nearestRaceId` is used only by `Kolco24App` for the startup prefetch (the on-tap prefetch in `MainActivity` uses the tapped `raceId` directly); `todayIso` is called by both `MainActivity` and `Kolco24App`.

**Config (secrets out of git):** `local.properties` keys `kolco24.apiBaseUrl` / `kolco24.appKeyId` / `kolco24.appSecret` → `BuildConfig` fields `API_BASE_URL` / `APP_KEY_ID` / `APP_SECRET` (see `app/build.gradle.kts`). Falls back to env `KOLCO24_API_BASE_URL` / `KOLCO24_APP_KEY_ID` / `KOLCO24_APP_SECRET` (so lint/test work in CI without the file); the build fails with a descriptive `error(...)` listing missing keys if neither is set. KSP + `kotlin.plugin.serialization` are applied on AGP 9's built-in Kotlin (do NOT apply `kotlin.android` on top — "plugin applied twice"); KSP version is pinned to `2.2.10-*`, and `android.disallowKotlinSourceSets=false` in `gradle.properties` lets KSP register generated sources.

**Scan overlay pattern:**
`MainActivity` wraps `Scaffold` in a `Box(Modifier.fillMaxSize())`. The `ScanScreen` is rendered after the Scaffold inside the same Box — when `showScan = true` it covers the entire screen. Dismiss via the close button or system back (handled by `BackHandler`) sets `showScan = false`. No navigation component used.

**Team-selection flow (enum overlay, same pattern):**
The flow reuses the scan-overlay approach instead of Navigation Compose. `MainActivity` holds three `rememberSaveable` fields: `teamFlowStep: TeamFlowStep { None, CompPicker, TeamPicker }` (a Kotlin enum is `Serializable`, no custom `Saver`), `pickerRaceId: Int?`, `confirmTeamId: Int?`. Overlays render after `ScanScreen` in the same `Box`: `CompPickerScreen` at `CompPicker`, `TeamPickerScreen` at `TeamPicker`, and `TeamSwitchSheet` (`ModalBottomSheet`) on top when `confirmTeamId != null`. A `BackHandler` (registered after the scan handler so it wins when enabled) steps the flow down: sheet (own dismiss) > TeamPicker > CompPicker > None. Confirming a team calls `teamRepository.selectTeam(...)` in `applicationScope` (not composition scope — the write must outlive the closing overlay) then collapses the whole flow.

**Settings overlay (same pattern):**
A fourth `rememberSaveable` flag `showSettings: Boolean` drives the `SettingsScreen` overlay. The «Команда» tab's «Настройки» row opens it (`onOpenSettings = { showSettings = true }`); it renders in the same `Box` **before** the CompPicker/TeamPicker/TeamSwitchSheet blocks so the picker draws on top when both are active. Its own `BackHandler(enabled = showSettings && teamFlowStep == None && confirmTeamId == null && !showScan)` closes it. «Сменить команду» inside Settings closes Settings then opens the comp picker (`showSettings = false; pickerRaceId = selectedRaceId; teamFlowStep = CompPicker`). `onScanClick` also resets `showSettings = false` so the scan FAB never opens behind a stale Settings overlay.
