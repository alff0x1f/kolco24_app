# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build commands

```bash
./gradlew assembleDebug              # build debug APK
./gradlew assembleRelease            # build release APK
./gradlew lintDebug                  # run lint (must pass before merging)
./gradlew testDebugUnitTest          # run unit tests
./gradlew connectedDebugAndroidTest  # run instrumented tests (requires emulator/device) — guards CheckpointDao preserve-on-resync + MIGRATION_1_2
```

## Architecture

Single-activity Jetpack Compose app (minSdk 24, targetSdk 36). No ViewModel, no Navigation Compose — screen state lives in `MainActivity.kt` via `rememberSaveable`. The data layer syncs races/teams/legend/member-tags via signed `GET` with ETag and uses Room as the single source of truth. NFC reader-mode and GPS-track recording are the two device-integration subsystems.

## Conventions & gotchas

These cross-cutting rules hold project-wide; the file bullets below assume them rather than repeating them.

- **minSdk 24, no desugaring** — use `SimpleDateFormat` (`Locale.US`, UTC where needed), never `java.time`. ISO date/time strings are fixed-width, so compare them **lexicographically** instead of parsing.
- **Pure models** (`ScanSession`, `CheckpointColor`, `ThemeMode`, `TrackProfile`, `TrustedClock`, `ProvisioningModel`, `ChipCheckModel`, `TrackModels`, `GpxExport`, the `*Logic`/mapper helpers, `LegendCrypto`) are **Android-free and JVM-unit-tested** (their `*Test` classes). **Compose UI, real Android adapters** (location engines, `TrackRecordingService`, the `NfcA`/DAO adapters), and trivial wiring are **untested by convention**. Trust boundaries (request signing, `ServerTimeInterceptor`) are the exception — they are tested.
- **Duplicate, don't couple** — small repeated UI rows (e.g. `SwitchTeamRow`/`ChangeTeamRow`, the marks vs track upload-status row) are **copied**, not extracted to a shared file.
- **Room is shipped** (see memory `room-released-with-migrations`) — on-device data must survive upgrades. A schema bump needs a real `Migration` appended to `.addMigrations(...)` **and** a committed `schemas/<n>.json`, or it crashes on upgrade. `exportSchema = true`.
- **Overlay pattern** — full-screen overlays (scan, settings, team-picker, admin, provisioning, check-chip, bind/unbind) are driven by `rememberSaveable` flags rendered after `Scaffold` in one `Box`, dismissed via `BackHandler`. No Navigation Compose. Per-team/race flows use the null-guarded keyed-`remember` pattern: `remember(selectedTeamId) { id?.let { repo.observe(it) } ?: flowOf(emptyList()) }`.
- **Writes outlive overlays** — a write triggered by a closing overlay (select team, bind, persist take) runs on `container.applicationScope`, not composition scope.
- **Repo refresh pattern** — on a `200`, persist rows (`replaceAllForRace`) **then** upsert the ETag as two separate transactions (a crash between them keeps fresh data with a stale ETag and self-heals next sync); the ETag upsert is skipped when the response omits the header.
- **Dual-target upload loop** (marks + track) — `Mutex.tryLock`-guarded, drains in `LIMIT` batches, marks only `accepted ∩ batch`, scoped by `(raceId, teamId)`, idempotent by client UUID, flushes to both cloud and LAN targets independently.
- **Secrets** — `BuildConfig` fields from `local.properties` keys (env-var fallback for CI); the build fails with a descriptive error if a required key is missing.

## Cross-file wiring (MainActivity)

- **Overlay stack** — all overlays render after `Scaffold` in one `Box`, each gated by a `rememberSaveable` flag (`showScan`, `showSettings`, `showAdmin`/`showProvisioning`/`showCheckChip`, `bindSlot`/`unbindSlot`, `teamFlowStep`/`confirmTeamId`). Sibling `BackHandler`s are registered in stacking order (later wins) and each fires only when no deeper overlay is up. Flags reset on `onScanClick` and on team switch (`LaunchedEffect(selectedTeamId)`). Theme/economy/admin-session are collected **once** here and threaded down as params (no second `collectAsState`).
- **Scan wiring** — the overlay is fed `scanBindings` (`nfcUid → numberInTeam`) **filtered to the current roster's slots** — a correctness guard: a stale binding for a removed member would otherwise let its chip falsely complete a take (`complete` only compares `present.size` to `expectedCount`). The FAB is gated on a resolved team (else routes to the picker); `onScanTag` refuses (`BadKp`) an empty roster. The opening tap and every later tap share one `ScanInput` path (`Captured` reuses the already-read `(code, uid)`, `Live` reads them) → `legendRepo.unlock` (if a code) → `classifyTag` → persist via `MarkRepository`. **`attachLocation` fires only on the new-take branch** (one-shot `currentLocationProvider.current()`, `null` = no-op).
- **Open-on-tap** — there is a **single open source**: the live-idle foreground path publishes `CapturedScan(code, uid, sample)` (no NDEF cold/warm launch — raw `K24` chips can't cold-launch the app). A `LaunchedEffect(captured, teamState)` dispatches it: another overlay up → drop; `Present` → open scan; `None`/`Missing` → open the picker; `Loading` → **return without clearing** so the next state transition re-fires. `ScanScreen` drains `pendingScan` so the opening tap is byte-identical to later taps. Auto-close on window expiry and on completion (a ~1 s green «Готово!» beat).

## UI layer

- `MainActivity.kt` — entry point; `HorizontalPager` swipe-tab navigation. **Hosts all overlay state** (see above) and the **NFC reader-mode infra** (`NfcAdapter.ReaderCallback`, enabled in `onResume`/disabled in `onPause`). A set of `@Volatile` tag hooks (`onTagScanned`, `onTagForChipInfo`, `onTagForMark`, `onTagForProvision`, `onTagForVerify`) are armed/cleared by each overlay's `DisposableEffect`; `onTagDiscovered` dispatches them by priority on the binder thread (hence `@Volatile`), and when none is armed the idle path reads `(code, uid)` once and publishes a `CapturedScan`. `nfcState: NfcState { NoHardware, Disabled, Available }` recomputed each `onResume`. `SelectedTeamState { None, Loading, Missing, Present }` drives the «Команда» tab (`Loading` suppresses the empty-state flash). `onRaceSelected` fires an on-tap prefetch of teams+legend on `applicationScope` so the network starts during the screen transition. A `LaunchedEffect` ticks `trustedClock.recomputeStatus()` every 5 s.
- `ui/marks/MarksScreen.kt` — Отметки tab on real take data; stateless. Pure `marksToTiles(marks, costOf, colorOf)` maps **one tile per `complete=true` take** (an incomplete take is kept for the server log but never shown — so an unbound roster shows nothing), oldest-first so new takes append; time from `trustedTakenAt ?: takenAt`. **Tile design:** the КП discipline color fills the **whole** flat square tile (no border/stripe/elevation) so a same-discipline cluster reads as one "blob"; an edge-to-edge 4-col grid with grout showing through 2dp seams. Pure `internal tileFill(color, darkTheme)`/`gridGrout(darkTheme)` carry a **muted, screen-scoped** palette distinct from the bright `CpColor*` bars; both take a plain `darkTheme: Boolean` and the caller resolves the **applied** theme via `MaterialTheme.colorScheme.surface.luminance() < 0.5f`, **not** `isSystemInDarkTheme()` (or a manual theme override would mismatch). `null`/unknown color → the only theme-flipping neutral fill (so a colorless tile never glows). Metrics from `takenPointCount`/`totalScore`; «ДО КВ» is a placeholder. **Empty state** is a state-aware next-step guide (no team / NFC off / NFC absent / chips unbound / ready), the ready branch adding a `LocationNudge` (when `!locationGranted`) and a pre-start `TrackNudge`. ## Data layer

- `Kolco24App.kt` — `Application`; builds `AppContainer`. **Launch A** (`onCreate`): `refreshRaces()` then prefetch the nearest race (`nearestRaceId`) — teams+legend+member-tags concurrently. **Launch B**: `collectLatest` the selected team and on each `raceId` refresh legend/teams/member-tags in a `supervisorScope` (a team switch cancels in-flight fetches), and fire `trackRepository.uploadAllPending()` + `markRepository.uploadAllPending()` so stranded points/takes opportunistically re-send. All on `applicationScope`.
- `AppContainer.kt` — manual DI (no Hilt). Builds the OkHttp client (`AppSignatureInterceptor` outer + `ServerTimeInterceptor` inner), `ApiClient` (cloud) + `localApiClient` (LAN, 3 s timeouts, **no** `ServerTimeInterceptor`), Room `AppDatabase`, all repositories, the preference stores, and `applicationScope`. **Construction-order cycles** (interceptor wants the bearer token / trusted time; repos want `ApiClient`) are broken by passing **lambdas** read only at request time (`tokenProvider = { adminAuthRepository.token() }`, `nowSeconds = { trustedClock.signingSeconds() }`). `cachedBootCount` is read **once per process** (`Settings.Global.BOOT_COUNT`). Public flows `trackRecordingState`, `trackUploadOutcomes`, `markUploadOutcomes` (the last two are **transient in-memory** per-target outcome maps — no Room table, no migration; self-heal after a restart). `clearDatabase()` (debug/test only) wipes all tables incl. ETags.
- `data/api/AppSignatureInterceptor.kt` — HMAC-SHA256 signs every request (canonical string per `docs/API.md`, 6 headers); `ts`/signature recomputed per call. Body hash by buffering the built request; a null/one-shot/unbufferable body falls back to `EMPTY_BODY_SHA256` (signing never crashes). `tokenProvider` adds the admin bearer; `nowSeconds` is the trusted-time `ts` source. **Self-heal retry-once (GET/HEAD only):** a `403` with a changed `ts` re-signs and retries once (the inner interceptor already re-anchored from the `403`'s `Date`) — **POST is never auto-retried** (a `403` is indistinguishable from a real auth failure; replay ≠ idempotent). Tested by `SigningTest`.
- `data/api/ApiClient.kt` — `FetchResult` (GET: `Success`/`NotModified`/`Forbidden`/`Error`) via one `conditionalGet(...)` with `If-None-Match`; `PostResult` (`Success`/`BadRequest`/`Unauthorized`/`Forbidden`/`Conflict`/`RateLimited`/`Offline`/`Error`) via one `post(...)` where **`parse` runs only on the 200/201 branch**. Methods: `fetchRaces`/`fetchTeams`/`fetchLegend`/`fetchMemberTags`; `login`/`logout`/`bindTag`/`uploadTrack`/`uploadMarks`. `uploadMarks` carries `source_install_id` in the **body** (unlike track). The same `uploadTrack`/`uploadMarks` serve both cloud and LAN — the target is the choice of `ApiClient` **instance**. `defaultOkHttpClient(...)` installs signature outer, optional `ServerTimeInterceptor` inner, no response `Cache` (every `Date` is live). Tested by `ApiClientTest`/`ApiClientMarksTest`/`TrackUploadTest`.
- `data/api/ServerTimeInterceptor.kt` — re-anchors `TrustedClock` from the HTTP `Date` header on every **network** response (free, backend-free server time). Pins the parsed epoch to an overflow-safe RTT-corrected midpoint; gated to network-only, RTT in `0..maxRttMs`, parseable `Date`. Trust is structural (installed only on the single `/app/` host). **Tested** (`ServerTimeInterceptorTest`) — a trust boundary, so the untested-adapters convention is waived.
- `data/api/dto/` — `@Serializable` snake_case DTOs, `ignoreUnknownKeys = true`. `RacesResponse`/`TeamsResponse` (note `start_number` is nullable/defaulted — undocumented); `LegendResponse` (legend always returned; a **locked** `CheckpointDto` carries an `enc:{iv,ct}` envelope **instead of** plaintext cost/description, so both nullable and `enc != null` is the locked sentinel; `color` is public, in both branches; `tags[].checkpoint_id` is the stable id); `MemberTagsResponse` (chip pool, `nfc_uid` already normalized server-side); `AuthDtos`/`TagDtos` (admin login + `TagBindResponse.code` = the hex written onto the chip); `TrackDtos`/`MarkDtos` (upload contracts in `docs/design/UPLOAD.md`). **Mark field names differ from `MarkEntity`** (`cp_nfc_uid ← cpUid`, `wall_ms ← takenAt`, `trusted_ms ← trustedTakenAt`, `elapsed_at ← elapsedRealtimeAt`); the pure `MarkEntity.toDto()` merges `present` with `presentDetails`, emitting a documented sentinel (`nfc_uid=null, number=0`) for a scored member with no snapshot so none is lost. Tested by `MarkDtoMappingTest`/`TrackUploadTest`.
- `data/db/` — Room **v2** (`exportSchema = true`, schemas committed). `MIGRATION_1_2` adds `marks.presentDetails` + `index_marks_raceId`; append future migrations with their own schema json (see the Room convention above). `MigrationTest` (instrumented) guards it. Tables: `RaceEntity` (also the domain model — no third layer), `SyncMetaEntity` (per-origin/resource ETag), `CategoryEntity` (`order` → `sortOrder`, reserved word), `TeamEntity` (`members` as a JSON column), `SelectedTeamEntity` (single row; `observe()` emits `null` = "no team chosen"), `CheckpointEntity` (cost/description **nullable** for locked CPs + `locked`/`encIv`/`encCt`/`color`; does **not** track `taken` — that's team-scoped, derived from marks), `TagEntity` (PK `(raceId, bid)`), `MemberTagEntity` (per-race chip pool, PK `(raceId, nfcUid)`), `MemberChipBindingEntity` (**local-only** member↔chip, PK `(teamId, numberInTeam)`), `MarkEntity` (**local-only** take log, PK = client UUID — snapshots КП meta, `present`/`presentDetails`, `complete`, the five `TimeSample` time columns, and the nullable `loc*` anti-cheat GPS columns), `TrackPointEntity` (**local-only** GPS log, PK = client UUID — `segmentId` per recording session, monotonic `elapsedRealtimeAt` + `bootCount`, back-projected `wallMs`, `trustedMs`). **DAO gotchas:** `CheckpointDao.replaceAllForRace` is **preserve-on-resync** (re-applies prior plaintext to a still-locked incoming id so a `200` never re-locks an offline-unlocked CP); `MarkDao.attachLocation` is a **column-scoped UPDATE** (not a full-row read-modify-write — it runs fire-and-forget in parallel with `addMember`, so a full write would lost-update) and **resets `uploaded*` to 0** to re-queue a row whose GPS fix landed after its first upload (server upserts by id → idempotent). Ordering is reboot-safe (`COALESCE(trustedMs, wallMs)`, then boot, then monotonic, then id). The JSON converters' round-trips are tested; `CheckpointDaoTest` (instrumented) guards preserve-on-resync.
- `data/RaceRepository.kt` / `TeamRepository.kt` / `LegendRepository.kt` / `MemberTagsRepository.kt` — per-resource sync repos following the **repo refresh pattern** (above); each stores its ETag in `sync_meta` under a resource key. `LegendRepository` also persists tags and owns `unlock(raceId, code): UnlockOutcome` — the **offline reveal path**: `bid` → tag lookup → `LegendCrypto.unlock(...)` on `Dispatchers.Default` → `Revealed` persists each CP via `checkpointDao.reveal(...)`. `TeamRepository` owns the selected team (`selectTeam`/`clearSelectedTeam`). `MemberTagsRepository` adds `hasBeenSynced(raceId)` (writes a synced-marker when the pool is served without an ETag) so the bind sheet distinguishes "not fetched" from "genuinely empty". Tested by `MemberTagsRepositoryTest`.
- `data/MemberChipBindingRepository.kt` — **local-only** member↔chip bindings (never uploaded). The warn+allow reassign is an atomic `@Transaction reassign` in the DAO (a chip is never momentarily on two slots); the decision stays in `decideBind`. **Binding-key assumption:** keyed by `(teamId, numberInTeam)` because a member exposes no stable participant id — relies on `number_in_team` being stable across `refreshTeams`; if the backend ever reorders slots a binding could re-associate (no mismatch detection). Tested by `MemberChipBindingRepositoryTest`.
- `data/MarkRepository.kt` — **local-only** take events with the dual-target upload loop (above). A take is **two-phase**: `startKpTake(...)` the moment the КП chip is scanned (fresh UUID, snapshots meta + roster size + buffered members + the five `TimeSample` times, upserts so it survives process death); `addMember(...)` adds each scanned member to `present` + `presentDetails` (set-semantics). `attachLocation(markId, fix)` is the fire-and-forget third phase (anti-cheat coordinate; `null` fix = no-op). Pure `takenPointCount`/`totalScore`/`takenPoints` derive the metrics + legend taken-set over distinct `complete` points (a repeat take is a new row, never double-counted). Tested by `MarkRepositoryTest`/`MarkRepositoryUploadTest`.
- `data/track/` — GPS track subsystem. `TrackModels.kt` (pure `RawFix`, `TrackPointLike`, `filterPoints` >50m accuracy drop for **read only**, reboot-safe `sortedTrackPoints`, upload-status value types + pure `uploadResultKind`); `GpxExport.kt` (pure GPX 1.1, **one `<trkseg>` per `segmentId`** so a stop→start gap isn't a teleport line); `TrackState.kt`; `TrackRepository.kt` (single source of truth; owns the impure `RawFix → TrackPointEntity` mapping — back-projects `wallMs`/`trustedMs` to the **fix moment**, not the insert time; dual batch upload independent per target; `uploadLoop` returns `UploadResultKind?` where an idle re-flush returns `null` so it can't overwrite a real error with a misleading "ok", and a `Success` with no forward progress hand-maps to `Error`); `TrackProfile.kt` (pure `Precise` 15 s / `Economy` 180 s) + `TrackProfilePreference.kt`; `LocationEngine.kt` (seam; pure `chooseEngineType`, untested Fused/Legacy adapters; `flush(onComplete)` forces buffered delivery before stop — fixes a «Стоп»-drops-last-batch bug); `CurrentLocationProvider.kt` + Fused/Legacy adapters (one-shot fresh fix for the take coordinate — **never throws** (any failure → `null`; "no coordinate" beats a stale one), freshness mandatory, drops fixes older than `MAX_FIX_AGE_MS`). Pure parts tested (`TrackPointMappingTest`/`UploadOutcomeTest`/`GpxExportTest`/`TrackProfileTest`/`LocationEngineFactoryTest`/`TrackRepositoryTest`).
- `data/nfc/MifareUltralightWriter.kt` — chip provisioning over a raw header-first on-chip format (no NDEF): page 4 = `K24` magic + packed version/type, pages 5–8 = the 16-byte code (`bid = sha256(code)[:16]`). **Gotchas:** writes the header **last** (commit-marker order) so an interrupted tap can't leave a valid header over a half-written code; reads back over the **same** open transport (only one `TagTechnology` connection allowed at a time); uses `NfcA` directly, **not** `MifareUltralight.get()`, because Android omits that tech for many NTAG/Ultralight chips; read tries FAST_READ then falls back to plain READs, never throwing. The `NfcTransport` fun-interface seam makes sequencing JVM-testable (`MifareUltralightWriterTest`); the thin `NfcA` adapter is not tested.
- `data/crypto/LegendCrypto.kt` — pure, offline, Android-free crypto for the encrypted legend (mirrors the server; spec in `docs/design/API.md`). `bid`, hand-rolled HKDF-SHA256, AES-256-GCM `open`, and `unlock(...)` doing the `bundle_blob → content_key → enc` indirection — **never throws** (catches → `Failed`). **Base64 is okio `ByteString`** (not `java.util.Base64` (API 26) or `android.util.Base64` (JVM-stubbed)). `LegendCryptoSanityTest` covers round-trips; `LegendCryptoTest` (server-vector assertions) is `@Ignore`d until a server fixture lands — the engine ships **crypto-unverified against the server** until then.
- `data/time/TrustedClock.kt` — pure trusted-time core. Derives **trusted** time from the monotonic `elapsedRealtime()` + server time (`trusted = serverEpochMs + (elapsedNow − anchorElapsedMs)`) — immune to wall-clock changes; only a **reboot** breaks it (detected by monotonic regression and/or `BOOT_COUNT` mismatch). `TimeSample` is one atomic snapshot of all time sources; `signingSeconds()` feeds request `ts`; `trustedAt(elapsedAt, bootAt)` extends the formula to an arbitrary past monotonic moment (the per-fix GPS use). **Warm start through boot identity** (`null == null` does NOT grant warm start — would falsely verify a reboot). Lock-free reads via `AtomicReference`; all writes under one lock; `persist` is best-effort (never throws on the OkHttp thread). Tested by `TrustedClockTest`.
- `data/AdminAuthRepository.kt` + `data/AdminTokenStore.kt` — race-admin session. `AdminSession { LoggedOut | LoggedIn }` seeded **synchronously** from the store (a past-expiry stored session → `LoggedOut`+cleared); `token()` is a synchronous `.value` read (so the OkHttp thread can call it). `login`/`logout` (logout always clears locally even offline)/`onUnauthorized`. Pure `isExpired` is a **lexicographic** UTC-string compare. The 30-day bearer is revocable, so plain prefs is acceptable. Tested by `AdminAuthRepositoryTest`/`AdminTokenStoreTest`.
- `data/ThemePreference.kt` / `data/TrackProfilePreference.kt` / `data/time/ClockAnchorStore.kt` / `data/InstallId.kt` — pure injected `load`/`save` stores + thin `fromSharedPreferences` adapters, read **synchronously** at construction (no cold-start flash). `ClockAnchorStore` writes the anchor as **one delimited key** (atomic — a kill mid-write can't leave a half-written mix). All pure parts unit-tested.
- `data/NfcUid.kt` — pure `normalizeNfcUid(raw: ByteArray): String` (per-byte zero-padded hex, uppercased, matching the server pool format). Tested by `NfcUidTest`.
- `data/DateUtils.kt` — `todayIso()` + `nearestRaceId(races, today)` (soonest current race), both `SimpleDateFormat`-based.
- `TrackRecordingService.kt` — foreground service (`type=location`, `START_NOT_STICKY` — a kill does not silently resume). Re-checks location permission first thing (TOCTOU on a revoke). Reads the active `TrackProfile`, mints one `segmentId` per recording session (pure `nextSegmentId`), batches `RawFix → trackRepository.insertAll`, and **flushes the buffer before stop** (`flushThen` runs `after` exactly once via callback or a ~4 s timeout fallback — fixes a «Стоп»-drops-last-batch bug). **Live-apply** profile toggle via a `profileJob` soft engine restart (no track gap). **Throttled live upload** piggybacks `trackRepository.uploadPending` + `markRepository.uploadPending` every `LIVE_UPLOAD_MIN_INTERVAL_MS` (10 min) via pure `shouldLiveUpload` (nullable "never uploaded" sentinel so the first batch always fires). **Guarantee:** «Стоп» + live-switch are lossless; a bare `onDestroy()`/hard kill may lose up to `maxDelay` (best-effort). Pure helpers tested (`SegmentIdTest`/`LiveUploadThrottleTest`); the service is not.
