# Data layer — per-file design notes

Per-file notes for `data/**`, `Kolco24App.kt`/`AppContainer.kt`, and `TrackRecordingService.kt`: the "why" behind each file, its gotchas, and test coverage. Read the relevant bullet before modifying a file here; update it when behavior changes. Cross-cutting rules live in `/CLAUDE.md` («Conventions & gotchas») and are assumed, not repeated.

## Data layer

- `Kolco24App.kt` — `Application`; builds `AppContainer`. **Launch A** (`onCreate`): `refreshRaces()` then prefetch the nearest race (`nearestRaceId`) — teams+legend+member-tags concurrently via `syncCoordinator.sourceFor(nearest)` (pin-aware: re-entering local mode after a process restart prefetches from LAN); also fires a best-effort `sweepOrphanPhotoDirs()` on startup (deletes `filesDir/marks/<id>/` directories whose markId has no live `MarkEntity` row — reclaims frames from killed mid-capture sessions). **Launch B**: `collectLatest` the selected team; inside a `supervisorScope` (a team switch cancels in-flight fetches), when `sourceFor(raceId) == Local` first `probeLocalAndRenew(raceId)` (heartbeat + immediate handback detection), then refresh legend/teams/member-tags via `sourceFor(raceId)` **re-read** after the probe (it may have just unpinned), and fire `trackRepository.uploadAllPending()` + `markRepository.uploadAllPending()` + `judgeScanRepository.uploadAllPending()` so stranded points/takes/piks opportunistically re-send. All on `applicationScope`.
- `AppContainer.kt` — manual DI (no Hilt). Builds the OkHttp client (`AppSignatureInterceptor` outer + `ServerTimeInterceptor` inner), `ApiClient` (cloud) + `localApiClient` (LAN, 3 s timeouts, **no** `ServerTimeInterceptor`), Room `AppDatabase`, all repositories, the preference stores, and `applicationScope`. **Construction-order cycles** (interceptor wants the bearer token / trusted time; repos want `ApiClient`) are broken by passing **lambdas** read only at request time (`tokenProvider = { adminAuthRepository.token() }`, `nowSeconds = { trustedClock.signingSeconds() }`). `cachedBootCount` is read **once per process** (`Settings.Global.BOOT_COUNT`). Public flows `trackRecordingState`, `trackUploadOutcomes`, `markUploadOutcomes` (the last two are **transient in-memory** per-target outcome maps — no Room table, no migration; self-heal after a restart). Owns the eager `scanFeedback: ScanFeedbackPlayer` (constructed in `onCreate` so its async `SoundPool.load()` is warm before the first tap), threaded down to the scan/bind/check/provision/judge-scan overlays and the idle path. `clearDatabase()` (debug/test only) wipes all tables incl. ETags **and** deletes `filesDir/marks/` so photo frames never outlive a DB wipe. **Local-mode-switch state:** `raceLeaseStore: RaceLeaseStore` (prefs-backed) seeds `raceLease: MutableStateFlow<RaceLease?>` synchronously at construction (a `StateFlow`, not a plain field, so the Settings switch collects live handback/expiry); `nowMs: () -> Long` prefers `trustedClock.trusted()` over wall clock; `isRacePinned: (raceId) -> Boolean` wraps `isPinned(raceLease.value, raceId, nowMs())` and is threaded into `TeamRepository`/`LegendRepository`/`MemberTagsRepository` (not `RaceRepository` — races are global, not race-scoped); `syncCoordinator: SyncCoordinator` binds all of the above plus the four repos' refresh calls as lambda seams (`fetchSync` collapses any `localApiClient.fetchSync` non-`Success` result to `null`). `judgeScanRepository: JudgeScanRepository` wires `cloudUploader = apiClient::uploadJudgeScans`, `localUploader = localApiClient::uploadJudgeScans`, `dao = database.judgeScanDao()`, `installId` — same lazy-by-lambda shape as the other uploaders.
- `data/api/AppSignatureInterceptor.kt` — HMAC-SHA256 signs every request (canonical string per `docs/API.md`, 6 headers); `ts`/signature recomputed per call. Body hash by buffering the built request; a null/one-shot/unbufferable body falls back to `EMPTY_BODY_SHA256` (signing never crashes). `tokenProvider` adds the admin bearer; `nowSeconds` is the trusted-time `ts` source. **Self-heal retry-once (GET/HEAD only):** a `403` with a changed `ts` re-signs and retries once (the inner interceptor already re-anchored from the `403`'s `Date`) — **POST is never auto-retried** (a `403` is indistinguishable from a real auth failure; replay ≠ idempotent). Tested by `SigningTest`.
- `data/api/ApiClient.kt` — `FetchResult` (GET: `Success`/`NotModified`/`Forbidden`/`Error`) via one `conditionalGet(...)` with `If-None-Match`; `PostResult` (`Success`/`BadRequest`/`Unauthorized`/`Forbidden`/`Conflict`/`RateLimited`/`Offline`/`Error`) via one `post(...)` where **`parse` runs only on the 200/201 branch**. Methods: `fetchRaces`/`fetchTeams`/`fetchLegend`/`fetchMemberTags`; `login`/`logout`/`bindTag`/`uploadTrack`/`uploadMarks`/`uploadMarkPhoto`/`uploadJudgeScans`. `uploadMarks`/`uploadJudgeScans` carry `source_install_id` in the **body** (unlike track). `post()` takes an optional `mediaType` (default `JSON_MEDIA_TYPE`); `uploadMarkPhoto(raceId, markId, frameId, bytes)` posts to `mark/<markId>/photo/<frameId>` with `IMAGE_JPEG_MEDIA_TYPE` and a `{ Unit }` parser (empty `200` body) — `frameId` is the frame's `<uuid>` filename stem, doubling as the server-side idempotency key; `404` (endpoint not deployed / mark row not landed) maps to `PostResult.Error` and self-heals on the next drain trigger. `uploadJudgeScans(raceId, sourceInstallId, scans)` posts to `judge_scans/` (no `team_id` — see `docs/design/UPLOAD.md`; server not yet implemented, so both targets return non-`Success` until it ships). The same `uploadTrack`/`uploadMarks`/`uploadMarkPhoto`/`uploadJudgeScans` serve both cloud and LAN — the target is the choice of `ApiClient` **instance**. `defaultOkHttpClient(...)` installs signature outer, optional `ServerTimeInterceptor` inner, no response `Cache` (every `Date` is live). Tested by `ApiClientTest`/`ApiClientMarksTest`/`TrackUploadTest`.
- `data/api/ServerTimeInterceptor.kt` — re-anchors `TrustedClock` from the HTTP `Date` header on every **network** response (free, backend-free server time). Pins the parsed epoch to an overflow-safe RTT-corrected midpoint; gated to network-only, RTT in `0..maxRttMs`, parseable `Date`. Trust is structural (installed only on the single `/app/` host). **Tested** (`ServerTimeInterceptorTest`) — a trust boundary, so the untested-adapters convention is waived.
- `data/api/dto/` — `@Serializable` snake_case DTOs, `ignoreUnknownKeys = true`. `RacesResponse`/`TeamsResponse` (note `start_number` is nullable/defaulted — undocumented); `LegendResponse` (legend always returned; a **locked** `CheckpointDto` carries an `enc:{iv,ct}` envelope **instead of** plaintext cost/description, so both nullable and `enc != null` is the locked sentinel; `color` is public, in both branches; `tags[].checkpoint_id` is the stable id; `totalCost`/`scoringCount` are top-level race aggregates — `scoringCount` is the count of CPs with `cost > 0` incl. locked, the denominator for the taken-КП counter, `default 0` for forward-compat before the server ships the field); `MemberTagsResponse` (chip pool, `nfc_uid` already normalized server-side); `AuthDtos`/`TagDtos` (admin login + `TagBindResponse.code` = the hex written onto the chip); `TrackDtos`/`MarkDtos`/`JudgeScanDtos` (upload contracts in `docs/design/UPLOAD.md`); `SyncDtos.kt` (`SyncManifestDto` — `race`/`data_source`/`lease_ttl_seconds`/`lease_expires_at`, fetched via `ApiClient.fetchSync(raceId)`, no ETag/304 by design; `versions` deliberately unmapped — see the lease bullet below). **Mark field names differ from `MarkEntity`** (`cp_nfc_uid ← cpUid`, `wall_ms ← takenAt`, `trusted_ms ← trustedTakenAt`, `elapsed_at ← elapsedRealtimeAt`); the pure `MarkEntity.toDto()` merges `present` with `presentDetails`, emitting a documented sentinel (`nfc_uid=null, number=0`) for a scored member with no snapshot so none is lost. `JudgeScanEntity.toDto()` mirrors the same field-rename shape (`wall_ms ← takenAt`, `trusted_ms ← trustedTakenAt`, `elapsed_at ← elapsedRealtimeAt`) but is a flat 1:1 copy — no `present`/snapshot merge, since a judge pik has no roster dimension. Tested by `MarkDtoMappingTest`/`TrackUploadTest`/`JudgeScanDtoTest`.
- `data/db/` — Room **v5** (`exportSchema = true`, schemas committed). `MIGRATION_1_2` adds `marks.presentDetails` + `index_marks_raceId`; `MIGRATION_2_3` (Phase 2, photo-mark upload) adds `marks.photosUploadedLocal`/`photosUploadedCloud` (`INTEGER NOT NULL DEFAULT 0`, per-target frame-drain flags); `MIGRATION_3_4` adds `legend_meta.scoringCount` (`INTEGER NOT NULL DEFAULT 0`, additive, no backfill — repopulates on the next legend refresh); `MIGRATION_4_5` (judge start/finish scan) creates `judge_scans` (nullable `bootCount`/`trustedTakenAt`, `NOT NULL` everything else) + `index_judge_scans_raceId` — column types/nullability and the index name must match Room's generated schema exactly, or `runMigrationsAndValidate` fails against `5.json`; append future migrations with their own schema json (see the Room convention above). `MigrationTest` (instrumented) guards all four. Tables: `RaceEntity` (also the domain model — no third layer), `SyncMetaEntity` (per-origin/resource ETag), `CategoryEntity` (`order` → `sortOrder`, reserved word), `TeamEntity` (`members` as a JSON column), `SelectedTeamEntity` (single row; `observe()` emits `null` = "no team chosen"), `CheckpointEntity` (cost/description **nullable** for locked CPs + `locked`/`encIv`/`encCt`/`color`; does **not** track `taken` — that's team-scoped, derived from marks), `LegendMetaEntity` (per-race legend aggregates the server computes and the client can't derive offline while locked: `totalCost` and `scoringCount`, the count of CPs with `cost > 0` incl. locked — see `LegendRepository`), `TagEntity` (PK `(raceId, bid)`), `MemberTagEntity` (per-race chip pool, PK `(raceId, nfcUid)`), `MemberChipBindingEntity` (**local-only** member↔chip, PK `(teamId, numberInTeam)`), `MarkEntity` (**local-only** take log, PK = client UUID — snapshots КП meta, `present`/`presentDetails`, `complete`, the five `TimeSample` time columns, the nullable `loc*` anti-cheat GPS columns, and the Phase-2 `photosUploadedLocal`/`photosUploadedCloud` flags), `TrackPointEntity` (**local-only** GPS log, PK = client UUID — `segmentId` per recording session, monotonic `elapsedRealtimeAt` + `bootCount`, back-projected `wallMs`, `trustedMs`), `JudgeScanEntity` (**local-only, write-once** judge start/finish pik, PK = client UUID, scoped by `raceId` **only** — no team dimension, since one judge station covers every team; `Index("raceId")` only, no unique constraint — repeat piks of the same participant are expected and each gets its own row). **DAO gotchas:** `CheckpointDao.replaceAllForRace` is **preserve-on-resync** (re-applies prior plaintext to a still-locked incoming id so a `200` never re-locks an offline-unlocked CP); `MarkDao.attachLocation` is a **column-scoped UPDATE** (not a full-row read-modify-write — it runs fire-and-forget in parallel with `addMember`, so a full write would lost-update) and **resets `uploaded*` to 0** to re-queue a row whose GPS fix landed after its first upload (server upserts by id → idempotent); `MarkDao.attachPhotos` (`updatePhotoPath`) is also a **column-scoped `@Transaction`** (only `photoPath`, `updatedAt`, and — Phase 2 — `photosUploadedLocal`/`photosUploadedCloud` change; `uploaded*` is **not** reset — `photoPath` is not in the marks DTO) and **resets `photosUploaded*` to 0** so appending frames re-queues the frame drain. **Phase 2 (photo upload):** the `method != 'photo'` filter is **dropped** from `unuploadedLocal`/`unuploadedCloud`/`pendingUploadScopes` — photo rows (empty `cpUid`/`cpCode`/`present`) now flow through the shared `/marks/` batch (server must partial-accept, see `docs/design/UPLOAD.md`); `uploadCounts` is truthful per target only when metadata **and** frames are done (`uploadedX AND (photoPath IS NULL OR photosUploadedX)`); `pendingUploadScopes` is widened to also return scopes with pending frames; `framePendingLocal`/`framePendingCloud(raceId, teamId, limit)` are the **method-agnostic** frame-drain candidate queries (`uploadedX=1 AND photosUploadedX=0 AND photoPath IS NOT NULL`, so an NFC take with attached photos drains identically to a standalone photo mark); `setPhotosUploadedLocalIfUnchanged`/`setPhotosUploadedCloudIfUnchanged(id, updatedAt)` are **version-guarded** flips (mirror `markUploadedCloudIfUnchanged`) so an `attachPhotos` racing mid-drain can't be clobbered back to `1` with a new frame unsent; `allIds()` backs the startup orphan sweep. **Upload-status page additions:** `uploadCountsMetadata(teamId, raceId): Flow<UploadCounts>` mirrors `uploadCounts` but drops the `photoPath`/`photosUploadedX` term (bare `uploadedLocal`/`uploadedCloud`) — answers "did the take row reach the server?" independent of its photos; `photoFrameRows(teamId, raceId): Flow<List<PhotoFrameRow>>` returns `(photoPath, photosUploadedLocal, photosUploadedCloud)` for in-scope rows where `photoPath IS NOT NULL`, folded by pure `MarkRepository.foldPhotoFrameCounts` into a frame-granular `UploadCounts`. Ordering is reboot-safe (`COALESCE(trustedMs, wallMs)`, then boot, then monotonic, then id). The JSON converters' round-trips are tested; `CheckpointDaoTest` (instrumented) guards preserve-on-resync; `MarkDaoTest` (instrumented) guards the frame-drain queries, `setPhotosUploaded*IfUnchanged`, `attachPhotos` column-scope/reset, and `uploadCountsMetadata`/`photoFrameRows`; `JudgeScanDaoTest` (instrumented) guards `unuploadedLocal`/`unuploadedCloud` scoping + ordering, `markUploadedLocal`/`markUploadedCloud`, and `pendingUploadRaces`.
- `data/lease/RaceLease.kt` + `RaceLeaseStore.kt` / `data/SyncSource.kt` / `data/sync/SyncCoordinator.kt` — the **local-mode-switch** lease/pin subsystem (race-day LAN pin; supersedes the abandoned lease-handoff plan, see memory `local-refresh-lease-plan`). `RaceLease(raceId, expiresAtMs)` pure model: `renewedLease` precedence `serverTtlSec` (clock-skew-immune) → `serverLeaseExpiresAtSec` (absolute, relies on a sane server clock) → `nowMs + DEFAULT_LEASE_MS` (12 h client default while the backend's lease is stubbed `null`); `isPinned(lease, raceId, nowMs)`; `applySyncResponse(manifest, raceId, nowMs): LeaseAction` (`Renew`/`Clear`/`Keep` — **never renews** on a `null` manifest, a manifest for a different race, or an unrecognized `data_source`). `RaceLeaseStore` copies the `ClockAnchorStore` shape: one delimited prefs key (`"$raceId|$expiresAtMs"`, atomic), malformed input → `null`. `SyncSource { Cloud, Local }` is the routing enum threaded through the four sync repos (default `Cloud`). `SyncCoordinator` is thin orchestration over lambda seams (the project's `tokenProvider` idiom — `SyncCoordinatorTest` injects fakes, no MockWebServer/DAO setup): `sourceFor(raceId)` (`Local` iff pinned); `probeLocalAndRenew(raceId)` (one LAN `fetchSync` → `applySyncResponse` → executes the `LeaseAction` — the single heartbeat/handback-detection primitive used at all three probe points: switch-on, pinned Launch B, pinned pull-to-refresh; **no periodic timer**); `enterLocalMode()`/`exitLocalMode(): LocalModeOutcome` (switch on/off — on: LAN reachable + `local` → pin + LAN fan-out; reachable + `cloud`/unrecognized → no pin + **Cloud** fan-out (a server disclaiming authority must not have its rows persisted); unreachable → nothing written, lease untouched; off: unconditional unpin + Cloud fan-out); `refreshAll(raceId)` (the PTR body — pinned → probe first, then fan-out via `sourceFor` **re-read** post-probe); `combineRefreshResults` folds a fan-out by explicit severity (`HttpError > Forbidden > Offline > Updated > NotModified > Skipped`). **Connectivity loss never releases the pin** — only expiry, server handback, or the switch does.
- `data/RaceRepository.kt` / `TeamRepository.kt` / `LegendRepository.kt` / `MemberTagsRepository.kt` — per-resource sync repos following the **repo refresh pattern** (above); each stores its ETag in `sync_meta` under a resource key. `LegendRepository` also persists tags and owns `unlock(raceId, code): UnlockOutcome` — the **offline reveal path**: `bid` → tag lookup → `LegendCrypto.unlock(...)` on `Dispatchers.Default` → `Revealed` persists each CP via `checkpointDao.reveal(...)`. `refreshLegend` upserts `LegendMetaEntity(raceId, response.totalCost, response.scoringCount)`; `totalCostForRace`/`scoringCountForRace(raceId): Flow<Int>` map `legendMetaDao.observeForRace(raceId)` to each aggregate (`?: 0`, so a cold cache or a pre-rollout server response reads as `0`). `TeamRepository` owns the selected team (`selectTeam`/`clearSelectedTeam`). `MemberTagsRepository` adds `hasBeenSynced(raceId)` (writes a synced-marker when the pool is served without an ETag) so the bind sheet distinguishes "not fetched" from "genuinely empty". **`SyncSource` routing:** `TeamRepository`/`LegendRepository`/`MemberTagsRepository`'s refresh methods take a `source: SyncSource = SyncSource.Cloud` param and an injected `isRacePinned` lambda — a `Cloud` call for a currently-pinned race returns `RefreshResult.Skipped` **both** at entry (before the network call) **and** re-checked in the `200` branch (an in-flight cloud response must not clobber fresher local rows just-pinned mid-flight); `RaceRepository.refreshRaces` takes the same `source` param but has **no** pin guard (races are a global list, not race-scoped) — it's only ever called with an explicit `Local` from inside `SyncCoordinator`. `RefreshResult` gained a `Skipped` case (`refreshErrorMessage` in `ui/common/PullToRefresh.kt` treats it as silent, same as `Updated`/`NotModified`). Tested by `MemberTagsRepositoryTest`.
- `data/MemberChipBindingRepository.kt` — **local-only** member↔chip bindings (never uploaded). The warn+allow reassign is an atomic `@Transaction reassign` in the DAO (a chip is never momentarily on two slots); the decision stays in `decideBind`. **Binding-key assumption:** keyed by `(teamId, numberInTeam)` because a member exposes no stable participant id — relies on `number_in_team` being stable across `refreshTeams`; if the backend ever reorders slots a binding could re-associate (no mismatch detection). Tested by `MemberChipBindingRepositoryTest`.
- `data/MarkRepository.kt` — **local-only** take events with the dual-target upload loop (above). A take is **two-phase**: `startKpTake(...)` the moment the КП chip is scanned (fresh UUID, snapshots meta + roster size + buffered members + the five `TimeSample` times, upserts so it survives process death); `addMember(...)` adds each scanned member to `present` + `presentDetails` (set-semantics). `attachLocation(markId, fix)` is the fire-and-forget third phase (anti-cheat coordinate; `null` fix = no-op). **Photo-mark additions:** `createPhotoMark(markId, cp, raceId, teamId, paths, expectedCount, sample)` — standalone photo take: `method="photo"`, `complete=true`, `present=[]`, `cpUid=""`; `markId` is minted by the entry point **before** the camera opens so frames write under `marks/<markId>/` before the DB row exists; `cost = cp.cost ?: 0` (0 while locked, real value after reveal); anti-cheat coordinate attached via the existing `attachLocation` path. `attachPhotos(markId, newPaths, now)` — column-scoped `@Transaction` that reads + merges + writes `photoPath`/`updatedAt`/`photosUploaded*` (**does not reset `uploaded*`** — `photoPath` is not in the marks DTO; **does reset `photosUploaded*`** to re-queue the frame drain). Pure `takenPointCount`/`totalScore`/`takenPoints` derive the metrics + legend taken-set over distinct `complete` points (a repeat take is a new row, never double-counted); `takenPointCount` also has a `(marks, costOf)` overload that counts only distinct `complete` takes with `costOf(it) > 0` — the scoring-only numerator behind `MarksScreen`'s «ВЗЯТО» and (via `isScoring()`/`scoringCountForRace`) `LegendScreen`'s `ScoreCard`. **Phase 2 frame upload:** `PhotoFrameUploader` (`(raceId, markId, frameId, bytes) -> PostResult<Unit>`, defaults to `Offline` — never a silent success) and `PhotoFrameReader` (`(relPath) -> ByteArray?`, defaults to `{ null }`) are injected seams, one uploader instance per target (cloud/local), mirroring `MarkUploader`. `flushScope` runs each target's metadata `uploadLoop` first, then its `frameDrainLoop` (method-agnostic — keys on `photoPath IS NOT NULL`, not `method='photo'`, so NFC takes with attached photos drain too): per mark in a `framePending*` batch, a **transient/target-wide** failure (`Offline`/`403`/`404`/`429`/`401`/`409`/`5xx`/other `Error`) stops the whole target for this trigger; a **hard per-frame** failure (`400`, or `413` → `Error(413)`, via `isHardFrameFailure`) or a **null read** (missing/unreadable file) leaves that mark pending and moves to the next mark (a poison frame never blocks later good marks — see the plan's "known tradeoff": a persistent poison frame or permanently-missing file keeps a mark visibly pending forever, by design); all-frames-accepted flips the mark via `setPhotosUploaded*IfUnchanged`; the loop terminates on no-progress (mirrors `uploadLoop`'s stuck-detection). The metadata and frame results per target are combined into **one** `onUploadOutcome` call via pure `combineOutcome(metadata, frame)` (precedence `Error > Offline > Ok > null`) so a frame `Ok` can never mask a metadata `Error`/`Offline`. **Upload-status page additions:** `uploadCountsMetadata(teamId, raceId)` delegates to the DAO of the same name; `photoFrameCounts(teamId, raceId): Flow<UploadCounts>` = `markDao.photoFrameRows(...).map(::foldPhotoFrameCounts)`; pure `foldPhotoFrameCounts(rows)` sums `photoPaths(r.photoPath).size` per row for `total`, gating `local`/`cloud` sums on each row's flag — a mid-drain mark (some frames sent, flag not yet flipped) contributes 0 to the numerators but its frames still count toward `total` (mark-granular tick, frame-granular denominator). Tested by `MarkRepositoryTest`/`MarkRepositoryUploadTest`/`PhotoFrameCountsTest`.
- `data/JudgeScanRepository.kt` — **local-only, write-once** judge start/finish pik log with the same dual-target (cloud + LAN) idempotent batch upload loop shape as `TrackRepository`, but scoped by `raceId` alone (no `teamId` — a judge station scans every team). `record(raceId, eventType, participantNumber, nfcUid, sample)` mints a fresh UUID and inserts immediately (mirrors `MarkRepository.startKpTake`'s `TimeSample` mapping) — there is no second phase; unlike a mark, a pik is never mutated after insert, so the DAO's `markUploaded*` needs no `updatedAt` version guard. `JudgeScanUploader` fun-interface seam (`(raceId, sourceInstallId, scans) -> PostResult<JudgeScanUploadResponse>`, defaults to `Offline`) mirrors `MarkUploader`/`TrackUploader`, one instance per target. `uploadPending(raceId)`/`uploadAllPending()` are both `Mutex.tryLock`-guarded (a losing concurrent call skips rather than double-POSTs); `uploadAllPending()` walks `JudgeScanDao.pendingUploadRaces()` so piks stranded under a race switch still flush. `flushRace` drains LAN then cloud **independently** via the shared `uploadLoop` (fetch `LIMIT=500` → upload → mark `accepted ∩ batch` → repeat; stops on an empty first fetch, a non-`Success` response, or no forward progress). Tested by `JudgeScanRepositoryTest` (accepted-subset marking, offline/error re-queue, dual-target independence, `tryLock` skip).
- `data/marks/PhotoPaths.kt` — pure codec for the JSON list of relative photo paths in the existing `marks.photoPath` TEXT column (migration-free — column type stays TEXT). `encodePhotoPaths`/`photoPaths` encode and decode; `photoPaths` filters each entry through a path-traversal guard (`marks/<markId>/<uuid>.jpg`, 3 segments, no `..`, no absolute prefix) so a `File(filesDir, relPath)` resolve can never escape `filesDir`. `thumbPathOf(framePath)` derives the tile thumbnail (`<uuid>.jpg` → `<uuid>.thumb.jpg`) — thumbs are a **filename convention, never a `photoPath` entry** (the upload drain sends every column entry to the server), derived at the render/delete site. Tested by `PhotoPathsTest`.
- `data/marks/PhotoTarget.kt` — pure router. `PhotoTarget { AttachTo(markId, cpNumber, checkpointId) | AskNumber }`. `decidePhotoTarget(marks, nowMs)` looks at the newest `complete` take's effective time (`trustedTakenAt ?: takenAt`); delta ≤ `PHOTO_ATTACH_WINDOW_MS = 180_000L` → `AttachTo`, else `AskNumber` (a future-timestamped take always attaches — negative delta is always ≤ window). `resolvePhotoCheckpoint(number, legend)` — locked CPs resolve normally (`locked=true`, `cost=null` is the core «сорвали метку» scenario). `filterCheckpointsByQuery(legend, query)` — case-insensitive substring on number or description. Tested by `PhotoTargetTest` / `ResolvePhotoCheckpointTest` / `FilterCheckpointsByQueryTest`.
- `data/marks/PhotoStorage.kt` — Android adapter for frame I/O. `writeDownscaledJpeg` decodes CameraX JPEG bytes, applies `rotationDegrees` (baked into pixels — re-encoding drops EXIF), downscales to `MAX_EDGE_PX = 1600`, saves to `filesDir/marks/<markId>/<uuid>.jpg`, then writes a sibling `<uuid>.thumb.jpg` tile thumbnail (`THUMB_MAX_EDGE = 512`, quality 75) **best-effort after the frame is safely on disk** — a failed thumb never fails the capture (the tile falls back to the full frame). `sweepOrphanDirs` reclaims directories whose markId has no `MarkEntity` row (thumbs go with the dir). `deletePhoto` validates the relative path before resolving against `filesDir` and deletes the frame **and its derived thumb**. Pure helpers (`scaledDimensions`, `orphanPhotoDirs`) JVM-tested by `PhotoStorageTest`; I/O adapter untested by convention.
- `data/ScanFeedbackPlayer.kt` — Android adapter playing the scan outcomes plus two celebration cues. Loads five clips in `init`: `beep_ok3`/`beep_err` (`success`/`failure`), `shutter` (photo-capture click), `checkpoint_mark_completed` (`checkpointCompleteFanfare()` fanfare), `mark_added_mario` (`coin()` tile pop-in). **Sound is for working chips only:** `success()`/`failure()` play a clip + vibrate `SUCCESS_VIBRATION_PATTERN`/`0,60,80,60`; `neutral()` (unknown chip) is **vibration-only — no sound**; `shutter()` is sound-only (no vibration, photo-mark only); `photoCaptureConfirm()` is vibration-only (`SUCCESS_VIBRATION_PATTERN`, the audible cue already fired at `shutter()`); `checkpointCompleteFanfare()` plays only the fanfare because the completing tap already played ordinary `success()` feedback; `coin()` is sound-only (a UI animation cue, not a chip confirmation). `play(kind: ScanFeedbackKind)` dispatches `feedbackFor`'s three outcomes only — `checkpointCompleteFanfare()`/`coin()`/`shutter()`/`photoCaptureConfirm()` are called directly by their call sites, not via `ScanFeedbackKind`. Owned by `AppContainer` and constructed **eagerly** in `Application.onCreate` so the asynchronous `SoundPool.load()` of all five clips (`USAGE_ASSISTANCE_SONIFICATION`, `maxStreams = 2` — fanfare and coin are sequenced by the celebration flow, never concurrent with each other) finishes before the first NFC tap; `SoundPool.play()` on an unloaded id returns 0 (silent no-op), so a missed sound on a very early tap is acceptable. **Vibration always fires regardless** — a missed vibration is not acceptable. `Vibrator` resolved via `VibratorManager` (API 31+) / `VIBRATOR_SERVICE` (below), each `vibrate(...)` split `VibrationEffect.createWaveform` (API 26+) / deprecated `vibrate(pattern, -1)` — every branch guarded by `Build.VERSION.SDK_INT` for `NewApi` lint. **`android.permission.VIBRATE` must stay in the manifest** or every vibrate is a silent no-op. `SoundPool.play`/`Vibrator.vibrate` are thread-safe → callable from the NFC binder thread and coroutines. **Untested by convention** (Android adapter).
- `data/track/` — GPS track subsystem. `TrackModels.kt` (pure `RawFix`, `TrackPointLike`, `filterPoints` >50m accuracy drop for **read only**, reboot-safe `sortedTrackPoints`, upload-status value types + pure `uploadResultKind`); `GpxExport.kt` (pure GPX 1.1, **one `<trkseg>` per `segmentId`** so a stop→start gap isn't a teleport line); `TrackState.kt`; `TrackRepository.kt` (single source of truth; owns the impure `RawFix → TrackPointEntity` mapping — back-projects `wallMs`/`trustedMs` to the **fix moment**, not the insert time; dual batch upload independent per target; `uploadLoop` returns `UploadResultKind?` where an idle re-flush returns `null` so it can't overwrite a real error with a misleading "ok", and a `Success` with no forward progress hand-maps to `Error`); `TrackProfile.kt` (pure `Precise` 15 s / `Economy` 180 s) + `TrackProfilePreference.kt`; `LocationEngine.kt` (seam; pure `chooseEngineType`, untested Fused/Legacy adapters; `flush(onComplete)` forces buffered delivery before stop — fixes a «Стоп»-drops-last-batch bug); `CurrentLocationProvider.kt` + Fused/Legacy adapters (one-shot fresh fix for the take coordinate — **never throws** (any failure → `null`; "no coordinate" beats a stale one), freshness mandatory, drops fixes older than `MAX_FIX_AGE_MS`). Pure parts tested (`TrackPointMappingTest`/`UploadOutcomeTest`/`GpxExportTest`/`TrackProfileTest`/`LocationEngineFactoryTest`/`TrackRepositoryTest`).
- `data/nfc/MifareUltralightWriter.kt` — chip provisioning over a raw header-first on-chip format (no NDEF): page 4 = `K24` magic + packed version/type, pages 5–8 = the 16-byte code (`bid = sha256(code)[:16]`). **Gotchas:** writes the header **last** (commit-marker order) so an interrupted tap can't leave a valid header over a half-written code; reads back over the **same** open transport (only one `TagTechnology` connection allowed at a time); uses `NfcA` directly, **not** `MifareUltralight.get()`, because Android omits that tech for many NTAG/Ultralight chips; read tries FAST_READ then falls back to plain READs, never throwing. The `NfcTransport` fun-interface seam makes sequencing JVM-testable (`MifareUltralightWriterTest`); the thin `NfcA` adapter is not tested.
- `data/crypto/LegendCrypto.kt` — pure, offline, Android-free crypto for the encrypted legend (mirrors the server; spec in `docs/design/API.md`). `bid`, hand-rolled HKDF-SHA256, AES-256-GCM `open`, and `unlock(...)` doing the `bundle_blob → content_key → enc` indirection — **never throws** (catches → `Failed`). **Base64 is okio `ByteString`** (not `java.util.Base64` (API 26) or `android.util.Base64` (JVM-stubbed)). `LegendCryptoSanityTest` covers round-trips; `LegendCryptoTest` (server-vector assertions) is `@Ignore`d until a server fixture lands — the engine ships **crypto-unverified against the server** until then.
- `data/time/TrustedClock.kt` — pure trusted-time core. Derives **trusted** time from the monotonic `elapsedRealtime()` + server time (`trusted = serverEpochMs + (elapsedNow − anchorElapsedMs)`) — immune to wall-clock changes; only a **reboot** breaks it (detected by monotonic regression and/or `BOOT_COUNT` mismatch). `TimeSample` is one atomic snapshot of all time sources; `signingSeconds()` feeds request `ts`; `trustedAt(elapsedAt, bootAt)` extends the formula to an arbitrary past monotonic moment (the per-fix GPS use). **Warm start through boot identity** (`null == null` does NOT grant warm start — would falsely verify a reboot). Lock-free reads via `AtomicReference`; all writes under one lock; `persist` is best-effort (never throws on the OkHttp thread). Tested by `TrustedClockTest`.
- `data/AdminAuthRepository.kt` + `data/AdminTokenStore.kt` — race-admin session. `AdminSession { LoggedOut | LoggedIn }` seeded **synchronously** from the store (a past-expiry stored session → `LoggedOut`+cleared); `token()` is a synchronous `.value` read (so the OkHttp thread can call it). `login`/`logout` (logout always clears locally even offline)/`onUnauthorized`. Pure `isExpired` is a **lexicographic** UTC-string compare. The 30-day bearer is revocable, so plain prefs is acceptable. Tested by `AdminAuthRepositoryTest`/`AdminTokenStoreTest`.
- `data/ThemePreference.kt` / `data/TrackProfilePreference.kt` / `data/time/ClockAnchorStore.kt` / `data/InstallId.kt` — pure injected `load`/`save` stores + thin `fromSharedPreferences` adapters, read **synchronously** at construction (no cold-start flash). `ClockAnchorStore` writes the anchor as **one delimited key** (atomic — a kill mid-write can't leave a half-written mix). All pure parts unit-tested.
- `data/NfcUid.kt` — pure `normalizeNfcUid(raw: ByteArray): String` (per-byte zero-padded hex, uppercased, matching the server pool format). Tested by `NfcUidTest`.
- `data/DateUtils.kt` — `todayIso()` + `nearestRaceId(races, today)` (soonest current race), both `SimpleDateFormat`-based.
- `TrackRecordingService.kt` — foreground service (`type=location`, `START_NOT_STICKY` — a kill does not silently resume). Re-checks location permission first thing (TOCTOU on a revoke). Reads the active `TrackProfile`, mints one `segmentId` per recording session (pure `nextSegmentId`), batches `RawFix → trackRepository.insertAll`, and **flushes the buffer before stop** (`flushThen` runs `after` exactly once via callback or a ~4 s timeout fallback — fixes a «Стоп»-drops-last-batch bug). **Live-apply** profile toggle via a `profileJob` soft engine restart (no track gap). **Throttled live upload** piggybacks `trackRepository.uploadPending` + `markRepository.uploadPending` every `LIVE_UPLOAD_MIN_INTERVAL_MS` (10 min) via pure `shouldLiveUpload` (nullable "never uploaded" sentinel so the first batch always fires). **Guarantee:** «Стоп» + live-switch are lossless; a bare `onDestroy()`/hard kill may lose up to `maxDelay` (best-effort). Pure helpers tested (`SegmentIdTest`/`LiveUploadThrottleTest`); the service is not.
