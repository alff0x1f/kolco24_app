# Плитка взятого КП — «Цветная сетка» (мультигонка)

Спека для Android-разработчика. Описывает один тайл (`ColorTile`) и сетку из направления **1 · Цветная сетка**.

Цвет КП = этап/дисциплина мультигонки. Цвет заливает всю плитку, поэтому группы этапов читаются «пятнами» без подписей.

---

## 1. Сетка (контейнер)

| Параметр | Значение |
|---|---|
| Раскладка | `GridLayout` / `RecyclerView` c `GridLayoutManager`, **4 колонки** |
| Промежуток между плитками | **2dp** (и по горизонтали, и по вертикали) |
| Внешние отступы сетки | left/right **14dp**, bottom **12dp**, top — 0 |
| Порядок элементов | по времени взятия (хронология сверху вниз) |

---

## 2. Плитка — геометрия

| Параметр | Значение |
|---|---|
| Форма | **квадрат**, `aspectRatio 1:1` (ширина = (ширина_сетки − 3×2dp) / 4) |
| Скругление | **0dp** (плитки стыкуются встык, без радиуса) |
| Клиппинг | `clipChildren = true` (время в углу обрезается по краю) |
| Контент | по центру (vertical + horizontal center) |

---

## 3. Цвета (токены этапов)

Заливка плитки — **диагональный градиент 157°** от основного цвета к затемнённому на −14 по RGB.

| Токен | Основной (solid) | Низ градиента (−14) | Подпись |
|---|---|---|---|
| `red` | `#CB4233` | `#BD3425` | Красный |
| `blue` | `#2F6CAE` | `#215EA0` | Синий |
| `green` | `#2E9E57` | `#209049` | Зелёный |
| `yellow` | `#C99A1E` | `#BB8C10` | Жёлтый |
| `purple` | `#7C5AC0` | `#6E4CB2` | Фиолетовый |

**Градиент:** `linear-gradient(157°, solid 0%, shade(solid,−14) 100%)`.
В Android: `GradientDrawable` с двумя цветами, `angle` подобрать (157° ≈ сверху-слева → снизу-справа; для `GradientDrawable` это `orientation = TL_BR` либо кастомный shader).

`shade(hex, −14)` = каждый из R/G/B минус 14, клампится в 0…255.

### Состояние «Без цвета» (neutral)
КП без привязки к этапу (`color` пустой).

| Параметр | Значение |
|---|---|
| Заливка | `linear-gradient(157°, #DCE2EA 0%, #C7D0DA 100%)` |
| Текст кода | `#161A1F` (INK) |
| Тень кода | `0 1px 0 rgba(255,255,255,0.5)` (светлая, «выпуклая») |
| Время | `rgba(86,96,106,0.8)` |

---

## 4. Тени плитки (boxShadow → elevation/inset)

CSS-тени для веба. На Android ближайший эквивалент:

**Цветная плитка:**
- inset `0 1px 1px rgba(255,255,255,0.28)` — верхний световой блик
- drop `0 1px 2px rgba(20,22,28,0.14)` — лёгкая посадочная тень (≈ elevation 1dp)

**Neutral плитка** (вдавленная):
- inset `0 1px 2px rgba(255,255,255,0.7)` (верхний блик)
- inset `0 −2px 6px rgba(38,46,58,0.14)` (нижняя тень)
- inset `0 0 0 0.5px rgba(60,60,67,0.12)` (хайрлайн-рамка)

Inset-тени в Android рисуются как наложенные drawable/слои в `LayerListDrawable` или кастомным рисованием — не критичны, можно упростить до плоской заливки + 1dp elevation.

---

## 5. Код КП (главный элемент)

Формат: **`{баллы}-{номер}`**, напр. `2-15` = 2 балла, КП №15. Дефис приглушён.

| Параметр | Значение |
|---|---|
| Шрифт | **JetBrains Mono**, weight **700** |
| Размер | **22sp** |
| Letter-spacing | **−0.8** (≈ −0.036em) |
| Line-height | 1.0 |
| Цвет (на цвете) | `#FFFFFF` |
| Цвет (neutral) | `#161A1F` |
| Тень текста (на цвете) | `0 1px 2px rgba(0,0,0,0.18)` |
| Тень текста (neutral) | `0 1px 0 rgba(255,255,255,0.5)` |
| Дефис | `opacity 0.5`, горизонтальный margin 1px |

Реализация дефиса: `SpannableString`, на символ `-` повесить `ForegroundColorSpan` с альфой 0.5 (50% от цвета кода).

## 6. Время взятия (вторичный элемент)

| Параметр | Значение |
|---|---|
| Позиция | **абсолютно**, правый-нижний угол: right **8dp**, bottom **6dp** |
| Шрифт | JetBrains Mono, weight **500** |
| Размер | **10.5sp** |
| Letter-spacing | +0.2 |
| Цвет (на цвете) | `rgba(255,255,255,0.78)` |
| Цвет (neutral) | `rgba(86,96,106,0.8)` |
| Формат | `HH:MM` |

## 7. Состояния

### Нажатие (ripple)
Все плитки имеют общий tap-слой (класс `.tap`):
- overlay поверх плитки, `border-radius: inherit`
- hover → `rgba(0,0,0,0.035)`
- **active (нажато)** → `rgba(0,0,0,0.09)`, transition `0.12s`

В Android: `foreground = ?attr/selectableItemItemBackground` (ripple) либо тёмный `rgba(0,0,0,0.09)` pressed-state — без скругления, на весь квадрат. `tap-highlight` отключён.

### Последний взятый КП (`recent: true`)
> ⚠️ **В «Цветной сетке» подсветки последнего КП НЕТ.** Поле `recent` есть в данных,
> но плитка `ColorTile` его не использует — никакого кольца не рисуется. Это сделано
> намеренно: на сплошной цветной заливке зелёное кольцо читалось бы плохо.
> Реализовывать подсветку recent в этом направлении **не нужно.**
